Obtaining and running Slogan

The latest development code base of Slogan can be obtained from GitHub:

$ git clone https://github.com/vijaymathew/slogan.git

The repository will be cloned to a directory named slogan. Use these commands to compile and install Slogan:

$ ./configure-platform
$ make
$ sudo make install
$ slogan

The last command will launch the Slogan REPL identified by the prompt:

slogan>

The REPL is where you enter Slogan expressions and evaluate them to get results.

(If you don't want to install Slogan, after `make', type `./src/slogan' to launch the REPL).

Now as you have a working Slogan installation, you may want to follow along the short tutorial below.

A short tutorial

If you have already programmed in a language like C, Java, Python or JavaScript, it is easy to get started in Slogan. The basic arithmetic operators are all the same:

> 3 + 4;
7
> 3 + 4 * 2;
11
> (3 + 4) * 2;
14
> (3 + 4) / 2;
7/2

The result of the expression (3 + 4) / 2 may look a little unusual. Dividing two integers will sometimes produce a rational number, which is a mathematically accurate representation of the result. Other languages may return a float or a real number in this case. A division operation in Slogan can be forced to return an inexact real number by making one of the operands a real number or by using the conversion function rational_to_real.

> (3 + 4) / 2.;
3.5
> rational_to_real ((3 + 4) / 2);
3.5

Global variables are declared using the var statement: 

> var three = 3;
> three + 4 * 2;
11

If we have to apply an operation to a sequence of numbers, it will be easier to package the numbers into a list and apply the operation across the elements in a single expression. The map function enables us to do this: 

> map (function (x) x * 2, [0, 1, 2, 3, 4]);
[0, 2, 4, 6, 8];
> map (function (x) x + x, [0, 1, 2, 3, 4]);
[0, 2, 4, 6, 8];
> map (function (x, y) x + y, [0, 1, 2, 3, 4], [100, 101, 102, 103, 104]);
[100, 102, 104, 106, 108]

As you might have figured out from the above code, list literals are enclosed in [] and the elements are separated by comma.

Two objects are compared for equality using the == operator. It returns true for objects that are represented by the same object in memory: 

> 10 == 10;
true
> 10 == 10.;
false
> 10.0 == 10.;
true

For comparing two objects that may have the same structural layout but represented by separate objects in memory, the is_equal function is used:

> var a = [0, 1, 2, 3, 4];
> var b = [0, 1, 2, 3, 4];
> a== b;
false
> is_equal (a, b);
true
> var sqr = function (x) x * 2;
> var add_self = function (x) x + x;
> map (sqr, a) == map (add_self, a);
false
> is_equal (map (sqr, a), map (add_self, a));
true

(You can find a more detailed tutorial in the "help" sub-directory in the code base.)
