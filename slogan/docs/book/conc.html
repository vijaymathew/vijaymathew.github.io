Concurrency and Parallelism

In the cocurrent model, a program is organized as a composition of several autonomous activities or tasks. A web server that handle thousands of clients at once is an example of a concurrent program. Cocurrency is also essential for programs that interact with their environment, e.g games and GUI programs.

Slogan support three styles of concurrent programming. The first and the most simpler style is declarative concurrency, which enable functional reasoning techniques to be applicable to concurrent programs. The second style of concurrent programming is based on the Actor model where the concurrent tasks work towards a common goal by doing independent computation and communicating with each other with asynchronous message passing. Slogan also support multi-tasking or multi-threading in the more traditional sense - using shared memory and locks. In the first section of this chapter, we cover all three concurrent programming styles.

Two concurrent tasks are executing parallely if they are scheduled to run on two separate hardware processors or cores. Two separate tasks may also be made to share a single core, by time-sharing. Depending on the implementation, a Slogan task may share computing resources with other tasks or it may run on an independent processor. The task abstraction is high-level and do not give the programmer much control on how the task is scheduled. To help programmers to take advantage of the multiple cores they may have on their machines, Slogan provide functions for executing a task on a dedicated processor-core. The parallel programming facilities of Slogan is also based on message-passing, which makes it much simpler to understand and use. Parallel programming will be the subject of the last section of this chapter.

8.1 Tasks

Units of concurrent program execution are known as tasks in Slogan. A new task is created and started by the task function. It takes a function as argument, which will be called from the newly launched task. The spawn operator (!) also starts a new task. The expression that follows the spawn will be evaluated in a new task. Both the task function and the spawn operator return an object that represents the new task. This object can be used to perform operations like sending asynchronous messages to the task, prematuarely terminating the task etc.

The task will finish running and its resources will be released once the function or expression used to start it is fully evaluated. If an error is thrown by during the execution of the task and it is unhandled within the task's body, then the task will be terminated immediately.

The next program defines a simple task that runs forever. It prints the message "hello" to the output stream, sleeps for 2 seconds and loops-back to print the message again. (If you are running this from the REPL, press Ctrl+C to terminate the process and get out of the loop).


task(^() let loop ()
         { showln("hello")
           task_sleep(2)
           loop() })
//> hello
    hello
    hello ...

The same program can be written using the spawn operator as shown below:


!let loop ()
 { showln("hello")
   task_sleep(2)
   loop() }
//> hello
    hello
    hello ...
