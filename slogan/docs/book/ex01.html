Processing Text Files

Part II of this book present complete Slogan programs that perform useful real-world tasks. These programs will also illustrate how to write good Slogan code.

The first program we study will demonstrate how to take textual data as input, do some processing on it and output the result. This program is based on the AWK programming language which is an important component of the UNIX programming environment. The utility we are going to develop will read "records" from an input file and process each record by calling a function defined by the user. A record is defined as a single line in the file, with each entry separated by spaces. As an example, the following listing shows a file (named emp.dat) with five records. Each line contains the name, hourly rate and total hours worked by an employee:


Mark   45  12
Susan  56  10
Matt   67  14
Jake   45   0
Jones  34  15

A report of total wages payable can be generated by writing a function like this:


function action(name, hourly_rate, hours_worked)
  when(hours_worked > 0)          
    showln(name, " ", hourly_rate * hours_worked)

To prepare the arguments for calling the action function, each line in the input file has to be processed as follows:

  1. Split the line by using whitespace as delimiter, into individual components.
  2. If component is in numeric format, convert that into a number, otherwise keep it as a string.
  3. If component is a boolean literal (true or false), convert that into the corresponding boolean value, otherwise keep it as a string.
  4. Apply action to these values.

The algorithm described above is captured by the handle_lines function shown below. It calls the utility parse_token to convert string tokens to numbers and boolean values.


function handle_lines()
  let (line = read_line(input))
    when (not(is_eof_object(line)))
    { apply(action, map(parse_token, string_split(line)))
      handle_lines() }

The parse_token function will check the first character of the token to decide whether to attempt a string to numeric conversion. If the conversion fails, the token itself is returned. If the token matches a boolean literal, the corresponding boolean value is returned. Otherwise, the token is returned unchanged:


function parse_token(token)
  let (c = string_at(token, 0))
    if (char_is_numeric(c) || c == \.)
      try string_to_number(token)
      catch (_) token
    else if (token == "true") true
    else if (token == "false") false
    else token

As we mentioned earlier, how to process each record is expressed by the action function which is specified by the user. How can we install this user specified function into our Awk program? One way is to let the user write this function as a Slogan script and pass this script's name as a command line argument to Awk. This script can then be dynamically loaded and evaluated. This will make sure that all definitions in the script will become available to the running program. The following code snippet shows how to get the script name from the command line, parse and evaluate it:


// ignore the first command line argument, which is the program's name.
let args = rest(command_line())

if (is_empty(args))
  error("expected script name as command-line argument")
else
  call_with_stream(file_reader(first(args)),
    ^(s)
     let (tokenizer = stream_tokenizer(false, s))
       eval(slogan(tokenizer)))

The only thing that remains to be done is to initialize the global variable input to point to the data file. This file's name is also accepted by the program as a command line argument. If no data file is specified by the user, input will default to the standard input.


args = rest(args)

let input = current_reader()
when (not(is_empty(args)))
  input = file_reader(first(args))

The full listing of the Awk program is available in the script awk.sn. This can be compiled into a stand-alone program by running the command:


$ slogan -x awk

User defined action functions can be written as separate scripts and fed to this program along with the data file to process. For example, we can save the original action function in a script called "myaction.sn" and run it on the "emp.dat" file.


// myaction.sn          
function action(name, rate_per_hour, total_hours)
  when (total_hours > 0)
    showln(name, " ", rate_per_hour * total_hours)    
      

$ ./awk myaction.sn emp.dat
//> Kathy 40.
   Mark 100.
   Mary 121.
   Susie 76.5
