22

Foreign Function Interface Reference

This chapter is a complete reference to the functions and macros defined in the C header file src/include/slogan.h. This header is the primary interface that allow function calls and type conversions between Slogan and C programs.

22.1 Configurable limits

The header file src/include/slogan.h define certain constants that control the number of arguments that can be passed to a C function and the number of C-struct proxies that can be defined in Slogan. These definitions are:


#define SLOGAN_LIBFFI_ARGC 32
#define SLOGAN_LIBFFI_STRUCT_DEFS 32
#define SLOGAN_LIBFFI_STRUCT_SIZE 1024

By default the number of arguments and struct proxies are limited at 32. A struct can be up to 1024 bytes in size. These limits can be increased or decreased as per the application requirements. The Slogan compiler must be rebuilt for these changes to take effect.

22.2 Low-level FFI

The lower-level FFI gives more control over making C function calls. It also allows C functions to callback into Slogan code.

A C function to be callable from low-level FFI must have one of the following signatures:


return_type f(___slogan_obj *args)
return_type f(void *argp, ___slogan_obj *args)

Args must be an array of ___slogan_obj objects. Argp is a pointer to an arbitrary C object and the return_type must be one of: void, void *, ___slogan_obj, char, char *, int, unsigned int, long, unsigned long, long long, float or double.

The type __slogan_obj represent all Slogan objects like strings, lists, integers and characters. The following section describes the C macros and functions defined in slogan.h for converting between ___slogan_obj and C types.
