Functional Data Structures

This chapter describes several techniques for designing functional data structures. These data structures, unlike their counterparts in imperative languages like C, do not depend in crucial ways on assignments, which are discouraged in functional programming.

Queues

Let us start by studying the implementation of a simple data structure - queue. A queue is a sequence of elements with an insert and a delete operation. The insert operation adds an element to one end of the queue and the delete operation removes an element from the other end. An additional operation front can also be defined to inspect the element at the end. We say the queue has FIFO (first-in, first-out) behavior. Let us investigate how to program queues in the functional world.

A naive queue implementation using lists is shown below:

slogan> function make_queue() []; 
slogan> function insert(q, x) [x q]; 
slogan> function delete(q) take(q, length(q) - 1); 
slogan> function front(q) at(q, length(q) - 1); 

slogan> define q = insert(insert(insert(make_queue(), 30), 20), 10); 
slogan> q;
=> [10, 20, 30]
slogan> front(q); 
=> 30
slogan> delete(q); 
=> [10, 20]

The problem with this implementation is that both delete and front are slow. They take time proportional to the length of the list. Is it possible to define delete and front to be of constant-time, like insert?. In strict functional style, i.e., without assignments, we cannot achieve it. The best we can do is to get amortized constant-time. That is, we can perform any sequence of n insert and delete operations in some constant time relative to n. Any individual operation might not be constant-time, however.

Here is the definition of a queue with amortized constant-time insert and delete:

slogan> function make_queue() [[] []]; 

slogan> function check(q) 
          match (q) 
            [[] r]: [reverse(r) []], 
            else: q; 

slogan> function insert(q, x) 
          match (q) 
            [f r]: check([f [x r]]);

slogan> function delete(q) 
          match (q) 
            [f r]: check([tail (f) r]);

slogan> function front(q) 
          match (q) 
            [f r]: head(f);

This implementation uses the pair (F R) to represent the queue. F and R are lists. F represents the front of the queue and R represents the back of the queue in reversed form. An element is inserted by adding it to the front of R and deleted by removing it from the front of F. For example, if F=[a b] and R=[d c], deleting the first element returns a and makes F=[b]. Inserting the element e makes R=[e d c]. Both operations are constant-time.

We move the elements in R to F when F becomes empty. This is is done by the check function which does a list reverse operation on R. The reverse takes time proportional to the length of R. Each element that goes through the queue is passed exactly once from R to F. Allocating the execution time of reverse to each element therefore gives a constant time per element, making the queue amortized.
