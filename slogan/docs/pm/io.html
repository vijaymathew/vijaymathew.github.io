Input and Output

All input and output operations are performed through objects called ports. A port is a pointer into a (possibly infinite) stream of data (often a file), an opening through which programs may draw bytes or characters from the stream or place bytes or characters into the stream. A port may be an input port, an output port, or both simultaneously.

There are initially three ports: the current input port, current output port, and current error port, which are textual ports connected to the process's standard input, standard output, and standard error streams. Several ways to open new ports are provided.

If one of the input operations is asked to read from a port that has reached the end of a finite stream, it returns the eof (end-of-file) object. The predicate is_eof_object may be used to determine if the value returned from the input operation is the eof object.

Ports are either binary or textual. A binary port allows a program to read or write 8-bit unsigned bytes, or "octets," from or to the underlying stream. A textual port allows a program to read or write characters.

A port may be buffered for efficiency, to eliminate the overhead of a call into the operating system for each byte or character. Three standard buffer modes are supported: block, line, and none. With block buffering, input is drawn from a stream and output is sent to the stream in chunks of some implementation-dependent size. With line buffering, buffering is performed on a line-by-line basis or on some other implementation-dependent basis. Line buffering is typically distinguished from block buffering only for textual output ports; there are no line divisions in binary ports, and input is likely to be drawn from a stream as it becomes available. With buffer-mode none, no buffering is performed, so output is sent immediately to the stream and input is drawn only as needed.

The remainder of this chapter covers operations on transcoders, file ports, standard ports, string, bytearray and network ports, and general port operations.

Transcoders

In many cases, the underlying stream is organized as a sequence of bytes, but these bytes should be treated as encodings for characters. In this case, a textual port may be created with a transcoder to decode bytes to characters (for input) or encode characters to bytes (for output). A transcoder encapsulates a codec that determines how characters are represented as bytes. These codecs are supported: ISO_8859_1, ASCII, UTF_8, UTF_16, UTF_16LE, UTF_16BE, UCS_2, UCS_2LE, UCS_2BE, UCS_4, UCS_4LE and UCS_4BE.

A transcoder also encapsulates an eol (end-of-line) style that determines whether and how line endings are recognized. If the eol style is none, no line endings are recognized. Supported eol styles are:

  • lf: line-feed character
  • cr: carriage-return character
  • crlf: carriage return followed by line feed

In addition to the codec and eol style, a transcoder encapsulates just one other piece of information: an error-handling mode that determines what happens if a decoding or encoding error occurs, i.e., if a sequence of bytes cannot be converted to a character with the encapsulated codec in the input direction or a character cannot be converted to a sequence of bytes with the encapsulated codec in the output direction. The error-handling mode is ignore, raise, or replace. If the error-handling mode is ignore, the offending sequence of bytes or the character is ignored. If the error-handling mode is raise, an exception with condition type i/o-decoding or i/o-encoding is raised; in the input direction, the port is positioned beyond the sequence of bytes. If the error-handling mode is replace, a replacement character or character encoding is produced: in the input direction, the replacement character is U+FFFD, while in the output direction, the replacement is either the encoding of U+FFFD for unicode codecs or the encoding of the question-mark character ( ? ) for other codecs.

function: transcoder(codec, @optional eol_style = !crlf, error_handling_mode = !ignore)
returns: a transcoder encapsulating codec, eol_style, and error_handling_mode.
function: transcoder_codec(transcoder)
function: transcoder_eol_style(transcoder)
function: transcoder_error_handling_mode(transcoder)
returns: the codec, eol_style, or error_handling_mode of transcoder.
slogan> define t = transcoder(!UTF_8, !lf, !raise); 
slogan> transcoder_codec(t); 
=> UTF_8
slogan> transcoder_eol_style(t); 
=> lf
slogan> transcoder_error_handling_mode(t); 
=> raise

File Ports

function: open_file_input_port(path, @optional options, bmode, tcoder)
returns: a new input port for the named file.

If present, options must be a list of symbols that control aspects of file operations. The only option that is valid for this function is truncate which controls whether the file will be truncated when it is opened. Bmode is a symbol that controls buffering. By default, buffering is turned off. If tcoder is present and not false, it must be a transcoder, and this function returns a textual input port whose transcoder is tcoder. Otherwise, this function returns a binary input port. 

function: open_file_output_port(path, @optional options, bmode, tcoder,  permissions = 0o666)
returns: a new output port for the named file.

If present, options must be a list of symbols that control aspects of file operations. The options that are valid for this function are create, no_create, maybe_create, append and truncate.

The create option requires that the file does not exist (otherwise an exception is raised) and creates the file when it is opened. The no_create option requires that the file exist (otherwise an exception is raised). The maybe_create option will create the file if it does not exist. This is the default option. The append option controls whether output will be added to the end of the file. This is useful for writing to log files that might be open by more than one process. By default appending is turned-off. The truncate option controls whether the file will be truncated when it is opened. This option is turned-on by default when no append option is provided and turned-off when append is not provided.

Bmode is a symbol that controls buffering. By default, buffering is turned off. If tcoder is present and not false, it must be a transcoder, and this function returns a textual output port whose transcoder is tcoder. Otherwise, this function returns a binary output port.

The permissions argument controls the UNIX permissions that will be attached to the file if it is created. The default value of this setting is 0o666.

function: open_file_input_output_port(path, @optional options, bmode, tcoder,  permissions = 0o666)
returns: a new input/output port for the named file.

Process Ports

The functions described in this section start a new operating system process and create a bidirectional port which allows communication with that process on its standard input and standard output. 

function: open_process_port(path, @key direction = !input_output, 
                            arguments = [], environment, 
                            directory, stdin_redirection = true, stdout_redirection = true, 
                            stderr_redirection, pseudo_terminal, show_console = true, 
                            transcoder)
returns: a port that can be used to communicate with an operating system process.

A descritpion of the arguments:

  • path: indicates the location of the executable program in the filesystem.
  • direction: should be one of the symbols - input_output, input or output. Defaults to input_output.
  • arguments: a list of strings. Indicates the string arguments that are passed to the program. The default value of this setting is the empty list (i.e. no arguments).
  • environment: a list of strings. Indicates the set of environment variable bindings that the process receives. Each element of the list is a string of the form "VAR=VALUE", where VAR is the name of the variable and VALUE is its binding. When environment is false, the process inherits the environment variable bindings of the Slogan program. The default value of this setting is false.
  • directory: indicates the current working directory of the process. When directory is false, the process uses the value of current_directory. The default value of this setting is false.
  • stdin_redirection: either true or false. Indicates how the standard input of the process is redirected. A setting of true will redirect the standard input from the process-port (i.e. what is written to the process-port will be available on the standard input). A setting of false will leave the standard input as-is, which typically results in input coming from the console. The default value of this setting is true.
  • stdout_redirection: either true or false. Indicates how the standard output of the process is redirected. A setting of true will redirect the standard output to the process-port (i.e. all output to standard output can be read from the process-port). A setting of false will leave the standard output as-is, which typically results in the output going to the console. The default value of this setting is true.
  • stderr_redirection: either true or false. Indicates how the standard error of the process is redirected. A setting of true will redirect the standard error to the process-port (i.e. all output to standard error can be read from the process-port). A setting of false will leave the standard error as-is, which typically results in error messages being output to the console. The default value of this setting is false.
  • pseudo_terminal: either true or false. This setting applies to UNIX. It indicates what type of device will be bound to the process' standard input and standard output. A setting of true will use a pseudo-terminal device (this is a device that behaves like a tty device even though there is no real terminal or user directly involved). A setting of false will use a pair of pipes. The difference is important for programs which behave differently when they are used interactively, for example shells. The default value of this setting is false.
  • show_console: either true or false. It controls whether the process' console window will be hidden or visible. The default value of this setting is true (i.e. show the console window).
  • transcoder: If transcoder is present and not false, it must be a transcoder, and this function returns a textual port whose transcoder is transcoder.
function: process_pid(port)
returns: the ID of the process attached to the process port port.
function: process_status(port)
returns: the exit status of the process attached to the process port port.

TCP Ports

function: open_tcp_server_port(address, @key port_number, 
                               backlog = 128, reuse_address = true, 
                               transcoder)
returns: a tcp-server-port from which network connections to clients are obtained.

A descritpion of the arguments:

  • address: indicates the internet address of the network interface on which connections requests are accepted, and possibly the IP port number. A string of the form "INTF:PORT" will be parsed and the value of INTF will be assigned to address and the integer value of PORT will be assigned to port_number. This argument can also be a 4 element u8 array which contains the 32-bit IPv4 address or an 8 element u8 array which contains the 128-bit IPv6 address.
  • port_number: must be an exact integer. Indicates the IP port number assigned to the socket which accepts connection requests from clients.
  • backlog: must be a positive-exact-integer. Indicates the maximum number of connection requests that can be waiting to be accepted by a call to read (technically it is the value passed as the second argument of the UNIX listen() function). The default value of this setting is 128.
  • reuse_address: either true or false. Controls whether it is possible to assign a port number that is currently active. Note that when a server process terminates, the socket it was using to accept connection requests does not become inactive immediately. Instead it remains active for a few minutes to ensure clean termination of the connections. A setting of false will cause an exception to be raised in that case. A setting of true will allow a port number to be used even if it is active. The default value of this setting is true.
  • transcoder: If transcoder is present and not false, it must be a transcoder, and this function returns a textual port whose transcoder is transcoder.

function: open_tcp_client_port(address, @key port_number, 
                               keep_alive, coalesce = true, 
                               transcoder)
returns: a tcp-client-port that represents a connection to a TCP server and allows 
         communication with that server.

A descritpion of the arguments:

  • address: indicates the internet address of the server, and possibly the IP port number. A string of the form "INTF:PORT" will be parsed and the value of INTF will be assigned to address and the integer value of PORT will be assigned to port_number. This argument can also be a 4 element u8 array which contains the 32-bit IPv4 address or an 8 element u8 array which contains the 128-bit IPv6 address.
  • port_number: must be an exact integer. Indicates the IP port number of the server to connect.
  • keep_alive: either true or false. Controls the use of the "keep alive" option on the connection. The "keep alive" option will periodically send control packets on otherwise idle network connections to ensure that the server host is active and reachable. The default value of this setting is false.
  • coalesce: either true or false. Controls the use of TCP's "Nagle algorithm" which reduces the number of small packets by delaying their transmission and coalescing them into larger packets. A setting of true will coalesce small packets into larger ones. A setting of false will transmit packets as soon as possible. The default value of this setting is true. Note that this setting does not affect the buffering of the port.
  • transcoder: If transcoder is present and not false, it must be a transcoder, and this function returns a textual port whose transcoder is transcoder.

Port Operations

function: is_port(obj)
function: is_output_port(obj)
function: is_input_port(obj)
returns: true if obj is a port, false otherwise.
function: close_port(port)
function: close_output_port(port)
function: close_input_port(port)
returns: void.

If port is not already closed, these functions closes it, first flushing any buffered bytes or characters to the underlying stream if the port is an output port. Once a port has been closed, no more input or output operations may be performed on the port. Because the operating system may place limits on the number of file ports open at one time or restrict access to an open file, it is good practice to close any file port that will no longer be used for input or output. Closing a port that has already been closed has no effect.

Closing a string or byte array port has no effect. As they consume no system resources, string and byte array ports can be safely reclaimed by the garbage collector.

function: port_position(port)
returns: the port's current position.

A port may allow queries to determine its current position in the underlying stream of bytes or characters. If so, port_position returns the current position. For binary ports, the position is always an exact nonnegative integer byte displacement from the start of the byte stream. For textual ports, the representation of a position is unspecified; it may not be an exact nonnegative integer and, even if it is, it may not represent either a byte or character displacement in the underlying stream. If port_position is called on a port that does not support it, an exception is raised.

function: port_has_port_position(port)
returns: true if port allows querying position, false otherwise.
function: set_port_position(port, pos, @optional whence = !beginning)
returns: the new position on success.

A port may allow its current position to be moved directly to a different position in the underlying stream of bytes or characters. If so, set_port_position changes the current position to pos. For binary ports, the position pos must be an exact nonnegative integer byte displacement from the start of the byte stream. For textual ports, the representation of a position is unspecified, but pos must be an appropriate position for the textual port. If set_port_position is called on a port that does not support it, an exception is raised.

The symbolic argument whence can be one of: beginning, current or end. When whence is omitted or is beginning, the position is relative to the beginning of the file. When whence is current, the position pos is relative to the current byte position of the file. When whence is end, the position pos is relative to the end of the file.

If port is a binary output port and the position is set beyond the current end of the data in the underlying stream, the stream is not extended until new data is written at that position. If new data is written at that position, the contents of each intervening position is unspecified. Binary ports created with open_file_output_port and open_file_input_output_port can always be extended in this manner within the limits of the underlying operating system. In other cases, attempts to set the port beyond the current end of data in the underlying object may result in an exception.

function: call_with_port(port, f)
returns: the value returned by the function f.

Call_with_port calls the function f with port as the only argument. If f returns, call_with_port closes the port and returns the values returned by f. If f raise an exception, call_with_port closes the port and re-throws the exception.

Call_with_port does not automatically close the port if a continuation created outside of f is invoked, since it is possible that another continuation created inside of f will be invoked at a later time, returning control to f.

returns: true if obj is the eof-object, false otherwise.

returns: the eof-object.
function: read_byte(@optional p = current_input_port()) 
returns: the next byte from binary input port p, or the eof object.
function: read_n_bytes(n, @optional p = current_input_port()) 
returns: an unsigned byte array containing the next n bytes from binary input port p, 
         or the eof object.
function: read_all_bytes(@optional p = current_input_port(), bufsize = 1024) 
returns: an unsigned byte array containing all the remaining bytes from binary input port p, 
         or the eof object.
function: read_char(@optional p = current_input_port()) 
returns: the next character from textual input port p, or the eof object.
function: peek_char(@optional p = current_input_port()) 
returns: the next character from textual input port p without actually removing it 
         from the stream, or the eof object.
function: read_n_chars(n, @optional p = current_input_port()) 
returns: a string containing the next n characters from textual input port p, or the eof object.
function: read_all_chars(@optional p = current_input_port(), bufsize = 1024) 
returns: a string containing all the remaining characters from textual input port p, 
         or the eof object.
function: read_line(@optional p = current_input_port(), separator = '\n', 
                    include_separator, max_length) 
returns: a string containing all the characters upto separator from the textual input port p, 
         or the eof object.

This function reads characters from the character input-port p until a specific separator or the end-of-file is encountered and returns a string containing the sequence of characters read. If it is specified, max_length must be a nonnegative exact integer and it places an upper limit on the number of characters that are read.

The separator is included at the end of the string only if it was the last character read and include_separator is true. The separator must be a character or false (in which case all the characters until the end-of-file are read). If it is not specified, p defaults to the current input-port. If it is not specified, separator defaults to '\n'. If it is not specified, include_separator defaults to false.

function: read_all(@optional p = current_input_port(), reader = read) 
returns: a list of values read by reader from the input port p.

This function repeatedly calls the function reader with the port p as the sole argument and accumulates a list of each value returned up to the end-of-file object. This function returns the accumulated list without the end-of-file object. If it is not specified, port defaults to the current input-port. If it is not specified, reader defaults to the function read.

function: read(@optional p = current_input_port()) 
returns: the next Slogan datum from the input port p, or the eof object.

The input port should point to a stream of compiled Slogan datums. (To parse and read tokens from Slogan source code, use the tokenizer functions described later in this chapter.)

Output Operations

function: write_chars(str, @optional p = current_output_port())

function: write_n_chars(str, start, end, @optional p = current_output_port()) 
returns: void
function: show(@key port = current_output_port(), obj1, ...)
function: showln(@key port = current_output_port(), obj1, ...)
returns: void

Writes the textual representation of the objects obj1 ... to the output-port port. Showln also outputs a line-feed character (newline) after printing the objects.

function: newline(@optional p = current_output_port()) 
returns: void

Outputs a line-feed character (newline) to the textual output-port p.

function: write(obj, @optional p = current_output_port()) 
returns: void

Writes the internal compiled representation of the Slogan object obj to the output-port p.

function: display(obj, @optional p = current_output_port()) 
returns: void

This function is similar to write but prints strings and characters found within obj directly. Strings are printed without quotation marks or escapes for special characters.

function: flush_output_port(@optional p = current_output_port()) 
returns: void

This function forces any bytes or characters in the buffer associated with the output-port p to be sent immediately to the underlying stream.

Tokenizers

A stream of characters read from an input port can be split into tokens using a tokenizer. This sections gives a description of functions used for tokenizing an input stream.

function: port_tokenizer(delimiters, @optional p = current_input_port()) 
returns: a tokenizer on the input port p.

If delimiters is either a character or a list of characters, a tokenizer will be returned that splits the stream at the charcter or each one of the characters. If delimiters is false, a tokenizer will be created that treats the input stream as Slogan source code and tokenizes the input based on the language syntax.

function: get_token(tokenizer) 
returns: the next token from tokenizer or the eof object.
function: peek_token(tokenizer) 
returns: the next token from tokenizer without removing it from the stream or the eof object.
function: slogan(tokenizer) 
returns: the next compiled expression from the Slogan source code tokenizer tokenizer 
         or the eof object.
