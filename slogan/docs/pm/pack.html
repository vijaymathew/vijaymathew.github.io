Packages

Slogan has a built-in package manager that makes it easy to work with packages developed by thrid-party developers. Any Slogan project can be converted into a package by adding just two files to its root folder - init.sn and build. Init.sn is a simple Slogan script that is called by the Slogan runtime for loading the package. So it should contain all initialization code for the package. Build should be an executable script that will compile the package. It can also contain code to install any dependencies that package might have. To make these points clear let us look at a simple example.

We will call our package "simple" and it will have the following directory structure:

simple/
        -> init.sn
        -> build
        -> src/
        --> core.sn

The file src/core.sn implements all the functionality provided by the package:

// src/core.sn

namespace simple;
function hello() "hello simple world!";
namespace;

init.sn finds the correct path to core.sn and simply loads it:

// init.sn
                
define load_path = string_append(slogan_root() "/packages/simple");
if (not(file_exists(load_path))) load_path = "."; // dev mode
link(string_append(load_path "/src/core"));

The build script is also equally simple - it just compiles init.sn into object code:

#!/bin/bash
slogan -c init

Make sure that the build script is executable:

$ chmod +x build

To test the package, first install it using the Slogan package manager:

$ slogan -i "simple,local,/mypackages/simple"

You should replace the path to the /mypackages/simple with an actual path on your machine. Now that the package is installed, you can load and use it from your REPL:

slogan> load_package("simple");
slogan> simple_hello(); 
=> hello simple world

A package can be removed from the system by using the -u option of the slogan command:

$ slogan -u simple

Package management functions

function: install_package(name, type, url, force)
returns: name if the package is successfully installed.

Install_package installs the package named name. Type must be a symbol and must be one of !local, !git or !remote. If type is !local, url should be the full path to the package directory or the full path of a tar.gz, a tar.bz2 or a .zip file that contains the package. For type !git, url must be the location of the git repository of the package. If type is !remote, url must point to a tar.gz, a tar.bz2 or a .zip package hosted on an HTTP or FTP server.

function: uninstall_package(name)
returns: name if the package is successfully removed.
function: load_package(name)
returns: name if the package is successfully loaded.
