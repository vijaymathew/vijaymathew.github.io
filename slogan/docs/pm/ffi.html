Foreign Function Interface

The Foreign Function Interface (FFI) is a mechanism for interfacing Slogan with C libraries. Slogan provides two interfaces to communicate with C libraries. First we will discuss the higher-level interface made available through the declare ffi statement.

The declare ffi statement has the following syntax:

declare ffi library_name list_of_prototypes

Here library name can be a string or a variable. If it is a string, it should be the full or relative path of the library to be loaded. If the path is relative, the library is first looked up in the local folder and then under the paths specified in the LD_LIBRARY_PATH environment variable. If library_name is a variable it should point to a library object loaded by the ffi_open function. (This function is described later in this document.)

List_of_prototypes contain prototype declarations of functions and structures in the shared library. The Slogan compiler uses these declarations to define functions that act as proxies to access these C objects.

A function prototype has the following syntax:

return_type c_function_name list_of_argument_types as proxy_function_name

The following table lists the types that can be used in a prototype and their equivalent C-types:

Note that the void type can only be used as a return type. It cannot be used as the type of a parameter.

The proxy_function_name is optional. If provided, the Slogan function that acts as a proxy will be bound to this name. Otherwise it will be bound to c_function_name.

As an example, two C functions and their corresponding prototypes are given below:

// sample_c_lib.c
#include <math.h>
double square_root(double f) { return sqrt(f); }
long add_li(int i, long l) { return ((long)i) + l; }
declare ffi "sample_c_lib.so" [double square_root [double] as sqrt,
                               long add_li [int, long]]

These prototype declarations mean that the Slogan program can call the C square_root function as sqrt(param) and the add_li function with the same name.

A structure prototype has the following syntax:

struct c_struct_name list_of_members

List_of_members is an association list of the types and field names. For example consider the definition of a C struct and its corresponding prototype:

// sample_c_lib.c
struct point {
   int x;
   int y;
};

struct point make_point(int x, int y)
{
   struct point p = {x, y};
   return p;
}
declare ffi "sample_c_lib.so" [struct point [int:x int:y],
                               point make_point [int, int]]

Note that a struct prototype is required only if Slogan has to call C functions that deal with struct instances directly, i.e not through pointers. If all C functions that you need to call deal with pointers to structures, you can get by with function declarations with the pointer type. Struct prototypes are also required if you want to inspect the contents of a pointer to a structure or you want to construct an instance of the structure from Slogan.

The FFI provides a number of functions for accessing and manipulating C structures:

function: c_struct_name(s)
returns: the name of the struct prototype as a symbol.
function: c_struct_get(s, m)
returns: the value of the member named m from the structure s, false if the member is not found.
function: c_struct_instance(n, v)
returns: a new instance of the structure named n with the members initialized to the values of the list v.

The values list should be specified in the same order as the members in the struct definition.

function: pointer_to_c_struct(n, ptr)
returns: converts the generic pointer ptr to the structure named n.

You can find a detailed sample on using the high-level FFI in the example/ffi folder of the source repository.

Configurable limits

The header file src/include/slogan.h defines certain constants that control the number of arguments that can be passed to a C function and the number of C-struct proxies that can be defined in Slogan. These definitions are:

#define SLOGAN_LIBFFI_ARGC 32
#define SLOGAN_LIBFFI_STRUCT_DEFS 32
#define SLOGAN_LIBFFI_STRUCT_SIZE 1024

By default the number of arguments and struct proxies are limited at 32. A struct can be up to 1024 bytes in size. These limits can be increased or decreased as per the applications requirement. Slogan should be rebuilt for changes in these limits to take effect.

Low-level FFI

The lower-level FFI gives more control over making C function calls. It also allows C functions to callback into Slogan code.

A C function to be callable from low-level FFI must have one of the following signatures:

  1. return_type f(___slogan_obj *args)
  2. return_type f(void *argp, ___slogan_obj *args)

Here args is an array of ___slogan_obj objects. Argp is a pointer to an arbitrary C object and the return_type is one of: void, void*, ___slogan_obj, char, char*, int, unsigned int, long, unsigned long, long long, float or double.

The type ___slogan_obj represents all Slogan objects like strings, lists, integers and characters. The header file slogan.h declares the functions and macros required for converting a ___slogan_obj to a C object and vice versa.

___charstring_to_slogan_obj initializes the ___slogan_obj s as a Slogan string with the characters from the C string c. ___charstring_to_slogan_obj creates a null-terminated string and ___nonnullcharstring_to_slogan_obj creates a string object that keeps track of its length. The C string c should always be null terminated.

The following C code shows the correct way to use these functions:

___slogan_obj hi(___slogan_obj *args)
{
  ___slogan_obj message;
  char *s = "hi from C!";
  char *p;
  ___slogan_obj_to_charstring(args[0], &p);
  printf("Slogan says \"%s\"\n", p);
      
  ___charstring_to_slogan_obj(s, &message);
  ___release_slogan_obj(message);
  return message;
}

The function hi can be invoked from Slogan as shown below, assuming that it has been compiled into a shared library called "test.so":

slogan> define clib = ffi_open("./test.so");
slogan> define f = ffi_fn(clib, "hi"); 
slogan> ffi_call_obj(f, #["hello from Slogan"]);
-> Slogan says "hello from Slogan"
=> hi from C!
slogan> ffi_close(clib); 
=> true
C macro:___pair(obj1, obj2)
returns: a ___slogan_obj that represents a Slogan pair with head and tail 
         pointing to obj1 and obj2.

Both obj1 and obj2 must be of types ___slogan_obj.

C macro:___head(p)
C macro:___tail(p)
returns: the head or tail objects of the ___slogan_obj p that represents a Slogan pair.
C constant:___TRU
C constant:___FAL
returns: the ___slogan_obj that represents the Slogan values true or false.
C constant:___NUL
returns: the ___slogan_obj that represents the empty list [].
C macro:___is_empty(pair)
returns: ___TRU if pair is the empty list [], ___FAL otherwise.

The following C function demonstrates how to receive a list of integers from Slogan and print it:

void print_i_list(___slogan_obj *args)
{
  ___slogan_obj a = args[0];
  while (!___is_empty(a)) {
    ___slogan_obj h = ___head(a);
    ___slogan_obj t = ___tail(a);
    int i;
    ___slogan_obj_to_int(h, &i);
    printf("%d\n", i);
    a = t;
  }
}

Print_i_list can be invoked from Slogan as:

slogan> ffi_call_void(pointer_to_print_i_list, #[[1, 10, -3, 99]]); 
-> 1
   10
   -3
   99
C macro:___body(array)
returns: a C array of ___slogan_obj objects that are the elements of the Slogan array array.

The following C function prints the first two elements of a Slogan array passed to it. The function expects the first element to be an integer and the second element to be a string.

void f(___slogan_obj *args)
{
  ___slogan_obj *b = ___body(args[0]);
  int i;
  char *s;
  ___slogan_obj_to_int(b[0], &i);
  ___slogan_obj_to_charstring(b[1], &s);
  printf("%d\n%s\n", i, s);
}

This function can be invoked from Slogan as:

slogan> ffi_call_void(pointer_to_f, #[#[100, "hello"]]); 
-> 100
   hello

Loading and Calling C Functions

This section describes the functions that let a Slogan program load a shared library and call the C functions exported by it.

function: ffi_open(libname) 
returns: an object representing a "handle" to the shared library file named libname, false 
         if the shared library could not be loaded.
function: ffi_close(lib_handle) 
returns: true if the shared library represented by lib_handle was successfully unloaded, 
         false otherwise.
function: ffi_fn(lib_handle, fnname) 
returns: returns the address of the function named fnname from the library represented by lib_handle.
         If the function is not found returns false.
function: ffi_call_void(fn_pointer, args) 
returns: void
function: ffi_call_void_pointer(fn_pointer, args) returns: a C pointer.
function: ffi_call_char_string(fn_pointer, args) 
returns: a string.
function: ffi_call_int(fn_pointer, args) 
returns: an integer.
function: ffi_call_uint(fn_pointer, args) 
returns: an integer.
function: ffi_call_int64(fn_pointer, args) 
returns: a 64-bit integer.
function: ffi_call_float(fn_pointer, args) 
returns: a real number.
function: ffi_call_double(fn_pointer, args) 
returns: a real number.
function: ffi_call_obj(fn_pointer, args) 
returns: an Slogan object.
function: ffi_call_void_with_void_pointer(fn_pointer, argp, args) 
returns: void
function: ffi_call_void_pointer_with_void_pointer(fn_pointer, argp, args) 
returns: a C pointer.
function: ffi_call_char_string_with_void_pointer(fn_pointer, argp, args) 
returns: a string.
function: ffi_call_int_with_void_pointer(fn_pointer, argp, args) 
returns: an integer.
function: ffi_call_uint_with_void_pointer(fn_pointer, argp, args) 
returns: an unsigned integer.
function: ffi_call_int64_with_void_pointer(fn_pointer, argp, args) 
returns: a 64-bit integer.
function: ffi_call_float_with_void_pointer(fn_pointer, argp, args) 
returns: a real number.
function: ffi_call_double_with_void_pointer(fn_pointer, argp, args) 
returns: a real number.
function: ffi_call_obj_with_void_pointer(fn_pointer, argp, args) 
returns: a Slogan object.

All the ffi_call_*** functions call the C function pointed to by fn_pointer by passing it the argument objects args and argp.Args must be an array and argp must be a pointer to a C object. That means argp will almost always be a value returned by a previous call to a C function that returns a void*.

Calling Slogan functions from C

The FFI implementation exports the ___call_fn function that enables C code to call any function implemented in Slogan. ___call_fn takes two arguments - the function object to call and its arguments. The arguments are passed as a single list. ___call_fn will return a ___slogan_obj which is the return value of the function object. The following example shows how to write a shared library that makes a call back into Slogan:

/* call_fn_test.c */
#include 
#include "slogan.h"

___slogan_obj call(___slogan_obj f)
{
  int error = 0;
  ___slogan_obj result;
  ___ON_THROW(result = ___call_fn(f, ___pair(___fix(100), ___pair(___fix(200), ___NUL))), error = 1);
  if (error == 1)
    {
      printf("Slogan function raised error\n");
      return ___FAL;
    }
  return result;
}

The ___ON_THROW macro will set the integer variable error to 1 if the Slogan function raises an exception. Once this is compiled into a shared library, call can be invoked as shown below:

slogan> define clib = ffi_open("./call_fn_test.so");
slogan> ffi_call_obj_1(clib, "call", add);
=> 300
