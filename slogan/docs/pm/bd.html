Built-in Datatypes

This chapter describes functions on datatypes that comes with Slogan, including lists, numbers, characters, strings, arrays, bytearrays, bitarrays, symbols, booleans, and hashtables. Before we cover the details of these functions, we will describe generic equivalence functions for comparing two objects and functions for determining the type of an object.

Equivalence and Type Predicates

function: is_eq(obj1, obj2) 
returns: true if obj1 and obj2 are identical, false otherwise.

Two objects are considered identical if they internally reference to the same memory location.

Is_eq cannot be used to compare numbers reliably. Slogan can use different objects internally for representing two big-integers that has the same value.

slogan> is_eq(!a, !b);
=> false
slogan> is_eq(!a, !a);
=> true
slogan> is_eq('a', 'a');
=> true
slogan> is_eq(10, 10); 
=> true
slogan> is_eq(38492948403840248034824037496494864917349749247947249749327482479, 
              38492948403840248034824037496494864917349749247947249749327482479); 
=> false

Is_eq is most often used to compare symbols or to check for pointer equivalence of allocated objects like pairs, arrays, and record instances.

function: is_eqv(obj1, obj2) 
returns: true if obj1 and obj2 are equivalent, false otherwise.

Is_eqv is similar to is_eq except that it return true for two numbers that are considered equal by the predicate number_is_eq. There is one exception for this rule though. Is_eqv will return false for is_eqv(+0.0, -0.0), whereas number_is_eq will return true for this comparison. This provides a way to distinguish between positive and negative zeros.

slogan> is_eqv(38492948403840248034824037496494864917349749247947249749327482479, 
               38492948403840248034824037496494864917349749247947249749327482479); 
=> true
function: is_equal(obj1, obj2) 
returns: true if obj1 and obj2 have the same structure and contents, false otherwise.

Two objects are equal if each of their constituent parts are equal according to is_eqv. The equals operator (==) has the same behavior as is_equal.

slogan> is_equal([1, 2, [3, 4]], [1, 2, [3, 4]]); 
=> true
slogan> [1, 2, [3, 4]] == [1, 2, [3, 4]]; 
=> true
slogan> is_equal("hello", "hello"); 
=> true
slogan> is_equal("hello", "hEllo"); 
=> false

Lists and Pairs

The pair is the most fundamental of all datastructures in Slogan. The most common use for pairs is to build lists, which are ordered sequences of pairs linked one to the next by the tail field. The elements of the list occupy the head fields of the pairs. The tail of the last pair in a proper list is the empty list, []. The tail of the last pair in an improper list can be anything other than [].

Pairs can also be used to build other datastructures like trees and simple lookup tables known as association lists.

function: pair(obj1, obj2)
function: cons(obj1, obj2) 
returns: a new pair whose head is obj1 and tail is obj2.
slogan> pair(3, 4);
=> [3 4]
slogan> pair(3, pair(4, 5)); 
=> [3 [4 5]]
slogan> pair(3, pair(4, pair(5, []))); 
=> [3, 4, 5]

These functions are defined as the composition of up to four cars and cdrs. The a's and d's between the c and r represent the application of car or cdr in order from right to left. For example, the procedure cadr applied to a pair yields the head of the tail of the pair.

Numbers

Slogan numbers are classified as integers, rational numbers, real numbers, or complex numbers. This classification is hierarchical, in that all integers are rational, all rational numbers are real, and all real numbers are complex.

A Slogan number may also be classified as exact or inexact, depending upon the quality of operations used to derive the number and the inputs to these operations. Most operations on numbers are exactness preserving: if given exact operands they return exact values, and if given inexact operands or a combination of exact and inexact operands they return inexact values.

Exact integer and rational arithmetic is supported to arbitrary precision; the size of an integer or of the denominator or numerator of a ratio is limited only by system storage constraints. Although other representations are possible, inexact numbers are represented by floating-point numbers supported by the host computer's hardware or by system software.

Slogan numbers are written in a straightforward manner not much different from ordinary conventions for writing numbers. An exact integer is normally written as a sequence of numerals preceded by an optional sign. For example, 3, +19, -100000, and 208423089237489374 all represent exact integers. Underscores (_) may be added to a numeric constant to increase readability. For example, the integer 150000 may be written as 1_50_000 or 150_000.

An exact rational number is normally written as two sequences of numerals separated by a slash (/) and preceded by an optional sign. For example, 3/4, -6/5, and 1/1208203823 are all exact rational numbers. A ratio is reduced immediately to lowest terms when it is read and may in fact reduce to an exact integer.

Inexact real numbers are normally written in either floating-point or scientific notation. Floating-point notation consists of a sequence of numerals followed by a decimal point and another sequence of numerals, all preceded by an optional sign. Scientific notation consists of an optional sign, a sequence of numerals, an optional decimal point followed by a second string of numerals, and an exponent; an exponent is written as the letter e followed by an optional sign and a sequence of numerals. For example, 1.0 and -200.0 are valid inexact integers, and 1.5, 0.034, -10e-10 and 1.5e-5 are valid inexact rational numbers. The exponent is the power of ten by which the number preceding the exponent should be scaled, so that 2e3 is equivalent to 2000.0.

The exactness of a numeric representation may be overridden by preceding the constant by either 0e or 0i. 0e forces the number to be exact, and 0i forces it to be inexact. For example, 1, 0e1, 1/1, 0e1/1, 0e1.0, and 0e1e0 all represent the exact integer 1, and 0i3/10, 0.3, 0i0.3, and 3e-1 all represent the inexact rational 0.3.

Numbers are written by default in base 10, although the special prefixes 0b (binary), 0o (octal), 0d (decimal), and 0x (hexadecimal) can be used to specify base 2, base 8, base 10, or base 16. For radix 16, the letters a through f or A through F serve as the additional numerals required to express digit values 10 through 15. For example, 0b10101 is the binary equivalent of 2110, 0o72 is the octal equivalent of 5810, and 0xC7 is the hexadecimal equivalent of 19910. Numbers written in floating-point and scientific notations are always written in base 10.

If both are present, radix and exactness prefixes may appear in either order.

Characters

Characters are atomic objects representing letters, digits, special symbols such as $ or -, and certain nongraphic control characters such as space and newline. Characters are written enclosing them in single quotes ('). The written character representation of the letter A, for example, is 'A'. Backslash is used as an escape for special characters. Some common examples are given below:

  • '\n' - newline
  • '\r' - carriage return
  • '\t' - tab
  • '\\' - backslash
  • '\b' - backspace
  • '\a' - alarm
  • '\v' - vertical tab
  • '\e' - escape
  • '\d' - delete

Any Unicode character may be written with the syntax '\xhh', '\uhhhh' or '\Uhhhhhhhh' where n consists of two, four or eight hexadecimal digits representing a valid Unicode scalar value.

Strings

Strings are sequences of characters and are often used as messages, character buffers, or containers for blocks of text. A string is written as a sequence of characters enclosed in double quotes, e.g., "hi there". A double quote may be introduced into a string by preceding it by a backward slash, e.g., "two \"quotes\" within". A backward slash may also be included by preceding it with a backward slash, e.g., "a \\slash". Various special characters can be inserted with other two-character sequences, e.g., \n for newline, \r for carriage return, and \t for tab. Any Unicode character may be inserted with the syntax \xhh, \uhhhh or \Uhhhhhhhh where h consists of two, four or eight hexadecimal digits and represents a valid Unicode scalar value.

Strings are indexed by exact nonnegative integers, and the index of the first element of any string is 0. The highest valid index for a given string is one less than its length.

Symbols

Symbols are used for a variety of purposes as symbolic names in Slogan programs. Strings could be used for most of the same purposes, but an important characteristic of symbols makes comparisons between symbols much more efficient. This characteristic is that two symbols with the same name are identical in the sense of is_eq. The reason is that Slogan makes sure that two symbols with the same sequence of characters are always stored in the same location in memory.

All characters valid in identifiers can be used in symbols. Symbol constants are written by prefixing identifiers with the exclamation mark (!).

function: string_to_symbol(string)
returns: a symbol whose name is string.
slogan> is_eq(!hello, string_to_symbol("hello")); 
=> true

returns: the name of symbol as a string.

Arrays

Arrays are more convenient and efficient than lists for some applications. Whereas accessing an arbitrary element in a list requires a linear traversal of the list up to the selected element, arbitrary array elements are accessed in constant time. The length of an array is the number of elements it contains. Arrays are indexed by exact nonnegative integers, and the index of the first element of any array is 0. The highest valid index for a given array is one less than its length.

As with lists, the elements of an array can be of any type, and a single array can hold more than one type of object.

An array literal looks similar to a list literal, except for the fact that the opening bracket is preceded by a hash (#). For example, an array consisting of the elements 1, 2, and 3 would be written #[1, 2, 3].

slogan> array_map(fn (x, y) x * y, #[1, 2, 3], #[4, 5, 6]); 
=> #[4, 10, 18]
function: array_for_each(f , arr1, ...)
returns: void

Array_for_each is similar to array_map but the function f is called for its side-effect.

slogan> array_for_each(fn (x, y) showln(x * y), #[1, 2, 3], #[4, 5, 6]); 
-> 4 
   10
   18

Bytearrays

Bytearrays are arrays of raw binary data. They are organized as a sequence of exact unsigned or signed 8-bit integers. The length of a bytearray is the number of 8-bit bytes it stores, and indices into a bytearray are always given as byte offsets.

Unsigned bytearray literals are written with the #u8 prefix and signed byte arrays are written with the #s8 prefix. For example, this is an unsigned bytearray literal: #u8[1, 10, 250] and this is a signed bytearray literal: #s8[-10, 90, -100]. An unsigned bytearray can contain integers in the range 0-255 and a signed bytearray can contain integers in the range -128-127.

This section describe the functions that operate on both signed and unsigned bytearrays. The functions that deal with unsigned values have the suffix or infix u8 and the functions that deal with signed values have the suffix or infix s8.

function: is_u8array(obj)

returns: an array of the unsigned or signed values.
function: make_u8array(d, @optional fill)
function: make_s8array(d, @optional fill)
returns: an array of unsigned or signed values of size d.

The dimension d should be an unsigned integer. If fill is provided, the array elements are initialized to that value. Fill defaults to 0.

function: u8array_length(arr)
function: s8array_length(arr)
returns: the number of elements in the bytearray arr.
function: u8array_at(arr, n)
function: s8array_at(arr, n)
returns: the nth element of the bytearray arr.
function: u8array_set(arr, n, b)
function: s8array_set(arr, n, b)
returns: void

Sets the nth element of the bytearray arr to b.

function: u8array_to_list(arr)
function: s8array_to_list(arr)
returns: a list with the elements of the bytearray arr.
function: list_to_u8array(lst)
function: list_to_s8array(lst)
returns: a bytearray with the elements of the list lst.
function: u8array_fill(arr, fill)
function: s8array_fill(arr, fill)
returns: void

Sets all elements in the bytearray arr to fill.

function: u8array_append(arr, @optional a1, ...)
function: s8array_append(arr, @optional a1, ...)
returns: a new bytearray created by appending a1 ... to arr.
function: u8array_copy(arr)
function: s8array_copy(arr)
returns: a new bytearray with all the elements of arr.
function: u8array_shrink(arr, n)
function: s8array_shrink(arr, n)
returns: void

Reduces the size of the bytearray arr to n.

function: subu8array(arr, start, end)
function: subs8array(arr, start, end)
returns: a new bytearray with the elements of arr from the index 
         start (inclusive) to end (exclusive).
function: subu8array_fill(arr, start, end, fill)

returns: void

Similar to u8array_fill or s8array_fill, but only the elements from the index start (inclusive) to end (exclusive) are replaced with fill.

This function replaces part of the contents of the bytearray dst_arr with part of the contents of the bytearray arr. It copies elements from arr, beginning with index start (inclusive) and ending with index end (exclusive) to dst_arr beginning with index dst_start (inclusive).

Bitarrays

A bitarray is a compact array of bit values.

Bitarray literals have the prefix #b. For example, this is a bitarray literal with 4 bits: #b[1, 0, 1, 1]

slogan> bit_array(1, 0, 0, 1, 1);

Hashtables

Hashtables represent sets of associations between arbitrary Slogan values. They serve essentially the same purpose as association lists but are typically much faster when large numbers of associations are involved.

function: make_hashtable(hash_f, eq_predicate, @optional size)
retruns: a new hashtable.

Hash_f and eq_predicate must be function. If size is provided, it must be a nonnegative exact integer indicating approximately how many elements the hashtable should initially hold. Hashtables grow as needed, but when the hashtable grows it generally must rehash all of the existing elements. Providing a nonzero size can help limit the amount of rehashing that must be done as the table is initially populated.

The new hashtable computes hash values using hash_f and compares keys using eq_predicate, neither of which should modify the hashtable. Eq_predicate should compare two keys and return false only if the two keys should be distinguished. Hash_f should accept a key as an argument and return a nonnegative exact integer value that is the same each time it is called with arguments that eq_predicate does not distinguish.

slogan> define h = make_hashtable(string_hash, string_is_eq); 
slogan> hashtable_set(h, "name", "vijay"); 
slogan> hashtable_at(h, "name"); 
=> "vijay"
function: make_eq_hashtable(@optional size)
function: make_eqv_hashtable(@optional size)
retruns: a new hashtable that uses is_eq or is_eqv for comparing keys.

An eq_hashtable or a eqv_hashtable employs a hash function based on object addresses. Their hash and equivalence functions are suitable for any Slogan object.

function: make_equal_hashtable(@optional size)
retruns: a new hashtable that uses is_equal for comparing keys.

An equal_hashtable uses the built-in equal_hash function which takes into consideration the current structure and contents of a key.

function: is_hashtable(obj)
retruns: true if obj is a hashtable, false otherwise.
function: eq_hash(obj)
function: eqv_hash(obj)
function: equal_hash(obj)
function: string_hash(str)
function: string_ci_hash(str)
function: symbol_hash(sym)
retruns: an exact nonnegative integer hash value.

These hash function are suitable for use with the appropriate predicate: is_eq for eq_hash and symbol_hash, is_eqv for eqv_hash, is_equal for equal_hash, string_is_eq for string_hash, and string_is_ci_eq for string_ci_hash. The hash values returned by equal_hash, string_hash, and string_ci_hash are typically dependent on the current structure and contents of the input values and are thus unsuitable if keys are modified while they have associations in a hashtable.

function: hashtable_set(ht , key, value)
retruns: void

Key should be an appropriate key for the hashtable's hash and equivalence functions. Value may be any Slogan object. Any existing association that key holds in the hashtable is replaced with obj.

function: hashtable_at(ht , key, @optional default)
retruns: the value associated with key in the hashtable ht.

Key should be an appropriate key for the hashtable's hash and equivalence functions. Default may be any Slogan object and defaults to false.

function: hashtable_contains(ht , key)
retruns: true if an association for key exists in the hashtable ht, false otherwise.
function: hashtable_update(ht , key, f, default)
retruns: void

Key should be an appropriate key for the hashtable's hash and equivalence functions. Default may be any Slogan object. The function f should accept one argument, should return one value, and should not modify the hashtable.

Hashtable_update applies f to the value associated with key in the hashtable, or to default if no value is associated with key. If f returns, key is associated with the value returned by f, replacing the old association, if any.

function: hashtable_size(ht)
retruns: the number of entries in the hashtable ht.
function: hashtable_copy(ht)
retruns: a new hashtable containing the same entries as the hashtable ht.
function: hashtable_keys(ht)
retruns: an array containing the keys of the hashtable ht.
function: hashtable_values(ht)
retruns: an array containing the values of the hashtable ht.
function: hashtable_entries(ht)
retruns: a pair that with two arrays: the first one with keys and the second one 
         with the values of the hashtable ht.
function: hashtable_for_each(f, ht)
retruns: void

For each entry in the hashtable ht, calls the function f by passing it the key and value as arguments.

slogan> define h = make_eq_hashtable(); 
slogan> hashtable_set(h, !name, "nemo"); 
slogan> hashtable_set(h, !age, 1); 
slogan> hashtable_for_each(fn (k, v) showln(k, ": ", v), h); 
-> name: nemo
   age: 1
