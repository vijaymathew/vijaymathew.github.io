Built-in Datatypes

This chapter describes functions on datatypes that comes with Slogan, including lists, numbers, characters, strings, arrays, bytearrays, bitarrays, symbols, booleans, and hashtables. Before we cover the details of these functions, we will describe generic equivalence functions for comparing two objects and functions for determining the type of an object.

Equivalence and Type Predicates

function: is_eq(obj1, obj2) 
returns: true if obj1 and obj2 are identical, false otherwise.

Two objects are considered identical if they internally reference to the same memory location.

Is_eq cannot be used to compare numbers reliably. Slogan can use different objects internally for representing two big-integers that has the same value.

slogan> is_eq(!a, !b);
=> false
slogan> is_eq(!a, !a);
=> true
slogan> is_eq('a', 'a');
=> true
slogan> is_eq(10, 10); 
=> true
slogan> is_eq(100000000000000000000000000000000000, 100000000000000000000000000000000000); 
=> false

Is_eq is most often used to compare symbols or to check for pointer equivalence of allocated objects like pairs, arrays, and record instances.

function: is_eqv(obj1, obj2) 
returns: true if obj1 and obj2 are equivalent, false otherwise.

Is_eqv is similar to is_eq except that it return true for two two numbers that are considered equal by is_number_eq. There is one exception for this rule though. Is_eqv will return false for is_eqv(+0.0, -0.0), whereas is_number_eq will return true for this comparison. This provides a way to distinguish between positive and negative zeros.

slogan> is_eqv(100000000000000000000000000000000000, 100000000000000000000000000000000000); 
=> true
function: is_equal(obj1, obj2) 
returns: true if obj1 and obj2 have the same structure and contents, false otherwise.

Two objects are equal if each of their constituent parts are equal according to is_eqv. The equals operator (==) has the same behavior as is_equal.

slogan> is_equal([1, 2, [3, 4]], [1, 2, [3, 4]]); 
=> true
slogan> [1, 2, [3, 4]] == [1, 2, [3, 4]]; 
=> true
slogan> is_equal("hello", "hello"); 
=> true
slogan> is_equal("hello", "hEllo"); 
=> false
function: is_boolean(obj) 
returns: true if obj is either true or false, false otherwise.
function: is_empty(obj) 
returns: true if obj is the empty list [], false otherwise.
function: is_pair(obj) 
returns: true if obj is a pair, false otherwise.
slogan> is_pair([]); 
=> false
slogan> is_pair([1]);
=> true
slogan> is_pair([1 2]);
=> true
slogan> is_pair([1, 2, !a]); 
=> true
function: is_number(obj) 
returns: true if obj is a number, false otherwise.
function: is_complex(obj) 
returns: true if obj is a complex number, false otherwise.
function: is_real(obj) 
returns: true if obj is a real number, false otherwise.
function: is_rational(obj) 
returns: true if obj is a rational number, false otherwise.
function: is_integer(obj) 
returns: true if obj is an integer, false otherwise.

These predicates form a hierarchy: any integer is rational, any rational is real, any real is complex, and any complex is numeric. 

function: is_char(obj) 
returns: true if obj is a character, false otherwise.
function: is_string(obj) 
returns: true if obj is a string, false otherwise.
function: is_array(obj) 
returns: true if obj is an array, false otherwise.
function: is_byte_array(obj) 
returns: true if obj is a byte-array, false otherwise.
slogan> is_byte_array(#[1, 2, 3]); 
=> false
slogan> is_byte_array(#b[1, 2, 3]); 
=> true
function: is_bit_array(obj) 
returns: true if obj is a bit-array, false otherwise.
function: is_symbol(obj) 
returns: true if obj is a symbol, false otherwise.
function: is_hashtable(obj) 
returns: true if obj is a hashtable, false otherwise.
function: is_function(obj) 
returns: true if obj is a function, false otherwise.
slogan> function f() fn () 100; 
slogan> is_function(f);
=> true
slogan> is_function(f()); 
=> true
slogan> is_function(f()()); 
=> false

Lists and Pairs

The pair is the most fundamental of datastructure in Slogan. The most common use for pairs is to build lists, which are ordered sequences of pairs linked one to the next by the tail field. The elements of the list occupy the head fields of the pairs. The tail of the last pair in a proper list is the empty list, []; the tail of the last pair in an improper list can be anything other than [].

Pairs can also be used to build other datastructures like trees and simple lookup tables known as association lists.

function: pair(obj1, obj2) 
returns: a new pair whose head is obj1 and tail is obj2.
slogan> pair(3, 4);
=> [3 4]
slogan> pair(3, pair(4, 5)); 
=> [3 [4 5]]
slogan> pair(3, pair(4, pair(5, []))); 
=> [3, 4, 5]
function: head(p) 
returns: the head of the pair p.
function: tail(p) 
returns: the tail of the pair p.
function: set_head(p, obj) 
returns: void.

Changes the head of the pair p to obj.

function: set_tail(p, obj) 
returns: void.

Changes the tail of the pair p to obj.

function: list(obj, ...) 
returns: a list of obj, ....
slogan> list(); 
=> []
slogan> list(1, 2, 3);
=> [1, 2, 3]
function: is_list(obj) 
returns: true is obj is a proper list, false otherwise.
function: length(lst) 
returns: the number of elements in the list lst.
function: at(lst, n) 
returns: the nth (zero based) element in the list lst.
function: nth_tail(lst, n) 
returns: the nth (zero based) tail of the list lst.
function: drop(lst, n) 
returns: a new list with n elements removed from lst.

If n is greater than the length of lst, [] is returned. If n is negative, the elements are dropped from the tail of lst.

slogan> drop(%[a, b, c, d, e], 3); 
=> [d, e]
slogan> drop(%[a, b, c, d, e], -3); 
=> [a, b]
function: take(lst, n) 
returns: a new list with n elements from lst.

If n is greater than the length of lst, all elements are returned. If n is negative, the elements are taken from the tail of lst.

slogan> take(%[a, b, c, d, e], 3); 
=> [a, b, c]
slogan> take(%[a, b, c, d, e], -3); 
=> [c, d, e]
function: append()
function: append(list1, ..., obj) 
returns: the concatenation of the lists.

Append returns a new list consisting of the elements of the first list followed by the elements of the second list, the elements of the third list, and so on. The new list is made from new pairs for all arguments but the last; the last (which need not be a list) is merely placed at the end of the new structure.

slogan> append();
=> []
slogan> append([1, 2], [3]); 
=> [1, 2, 3]
slogan> append([1, 2], 3);
=> [1 [2 3]]
function: reverse(lst)
returns: a new list with the elements of lst in the reverse order.
