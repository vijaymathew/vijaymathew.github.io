Built-in Datatypes

This chapter describes functions on datatypes that comes with Slogan, including lists, numbers, characters, strings, arrays, bytearrays, bitarrays, symbols, booleans, and hashtables. Before we cover the details of these functions, we will describe generic equivalence functions for comparing two objects and functions for determining the type of an object.

Equivalence and Type Predicates

function: is_eq(obj1, obj2) 
returns: true if obj1 and obj2 are identical, false otherwise.

Two objects are considered identical if they internally reference to the same memory location.

Is_eq cannot be used to compare numbers reliably. Slogan can use different objects internally for representing two big-integers that has the same value.

slogan> is_eq(!a, !b);
=> false
slogan> is_eq(!a, !a);
=> true
slogan> is_eq('a', 'a');
=> true
slogan> is_eq(10, 10); 
=> true
slogan> is_eq(100000000000000000000000000000000000, 100000000000000000000000000000000000); 
=> false

Is_eq is most often used to compare symbols or to check for pointer equivalence of allocated objects like pairs, arrays, and record instances.

function: is_eqv(obj1, obj2) 
returns: true if obj1 and obj2 are equivalent, false otherwise.

Is_eqv is similar to is_eq except that it return true for two two numbers that are considered equal by number_is_eq. There is one exception for this rule though. Is_eqv will return false for is_eqv(+0.0, -0.0), whereas number_is_eq will return true for this comparison. This provides a way to distinguish between positive and negative zeros.

slogan> is_eqv(100000000000000000000000000000000000, 100000000000000000000000000000000000); 
=> true
function: is_equal(obj1, obj2) 
returns: true if obj1 and obj2 have the same structure and contents, false otherwise.

Two objects are equal if each of their constituent parts are equal according to is_eqv. The equals operator (==) has the same behavior as is_equal.

slogan> is_equal([1, 2, [3, 4]], [1, 2, [3, 4]]); 
=> true
slogan> [1, 2, [3, 4]] == [1, 2, [3, 4]]; 
=> true
slogan> is_equal("hello", "hello"); 
=> true
slogan> is_equal("hello", "hEllo"); 
=> false

Lists and Pairs

The pair is the most fundamental of datastructure in Slogan. The most common use for pairs is to build lists, which are ordered sequences of pairs linked one to the next by the tail field. The elements of the list occupy the head fields of the pairs. The tail of the last pair in a proper list is the empty list, []; the tail of the last pair in an improper list can be anything other than [].

Pairs can also be used to build other datastructures like trees and simple lookup tables known as association lists.

function: pair(obj1, obj2) 
returns: a new pair whose head is obj1 and tail is obj2.
slogan> pair(3, 4);
=> [3 4]
slogan> pair(3, pair(4, 5)); 
=> [3 [4 5]]
slogan> pair(3, pair(4, pair(5, []))); 
=> [3, 4, 5]

returns: the head of the pair p.
function: tail(p) 
returns: the tail of the pair p.

returns: void.

Changes the head of the pair p to obj.

function: set_tail(p, obj) 
returns: void.

Changes the tail of the pair p to obj.

returns: a list of obj, ....
slogan> list();

returns: a new list with n elements removed from lst.

slogan> drop(%[a, b, c, d, e], 3); 
=> [d, e]
slogan> drop(%[a, b, c, d, e], -3); 
=> [a, b]

slogan> take(%[a, b, c, d, e], -3);

The function f should accept one argument and return a single value. It should not modify lst. The elements of the returned lists appear in the same order as they appeared in the original list.

slogan> head(partition(is_odd, [1, 2, 3, 4, 5]));
=> [1, 3, 5]
slogan> tail(partition(is_odd, [1, 2, 3, 4, 5]));
=> [2, 4]

The function f should accept one argument and return a single value. It should not modify lst. If a program must distinguish between finding false in the list and finding no element at all, memp should be used instead.

slogan> find(is_even, [1, 2, 3, 4, 5]); 
=> 2
slogan> find(is_even, [1, 3, 5]); 
=> false
function: assq(obj, lst) 
function: assv(obj, lst) 
function: assoc(obj, lst) 
returns: first element of lst whose head is equivalent to obj, or false.

The argument lst must be an association list. An association list is a proper list whose elements are key-value pairs of the form [key value]. Associations are useful for storing information (values) associated with certain objects (keys).

The equivalence test for assq is is_eq, for assv is is_eqv, and for assoc is is_equal.

slogan> assq(!b, %[[a 1], [b 2], [c 3]]); 
=> [b 2]
slogan> assoc("b", %[["a" 1], ["b" 2], ["c" 3]]);
=> ["b" 2]
function: assp(f, lst) 
returns: the first element in lst for whose head f returns true, false otherwise.

The argument lst must be an association list. The function f should accept one argument and return a single value. It should not modify lst. 

slogan> assp(is_odd, %[[1 a], [2 b]]);
=> [1 a]
slogan> assp(is_even, %[[1 a], [2 b]]);
=> [2 b]

function: sort(lst, @optional test, type) 
returns: a list containing the elements of lst sorted according to test.

Test should be a function that expects two arguments and returns true if its first argument must precede its second in the sorted list. Duplicate elements are not removed. The default value of test is number_is_lt which compares two numeric values. Type specifies the sorting algorithm to use. In the current implementation type can take two values: !quick for "quick-sort" and !merge for "merge-sort".

slogan> sort([3, 4, 2, 1, 2, 5]); 
=> [1, 2, 2, 3, 4, 5]
slogan> list_to_string(sort(string_to_list("hello"), is_char_lt)); 
=> "ehllo"
function: quicksort(lst, @optional test)

returns: a list containing the elements of lst sorted according to test.

These functions are internally used by sort to get its work done.

Numbers

Slogan numbers are classified as integers, rational numbers, real numbers, or complex numbers. This classification is hierarchical, in that all integers are rational, all rational numbers are real, and all real numbers are complex.

A Slogan number may also be classified as exact or inexact, depending upon the quality of operations used to derive the number and the inputs to these operations. Most operations on numbers in are exactness preserving: if given exact operands they return exact values, and if given inexact operands or a combination of exact and inexact operands they return inexact values.

Exact integer and rational arithmetic is supported to arbitrary precision; the size of an integer or of the denominator or numerator of a ratio is limited only by system storage constraints. Although other representations are possible, inexact numbers are represented by floating-point numbers supported by the host computer's hardware or by system software.

Slogan numbers are written in a straightforward manner not much different from ordinary conventions for writing numbers. An exact integer is normally written as a sequence of numerals preceded by an optional sign. For example, 3, +19, -100000, and 208423089237489374 all represent exact integers. Underscores (_) may be added to a numeric constant to increase readability. For example, the integer 150000 may be written as 1_50_000 or 150_000.

An exact rational number is normally written as two sequences of numerals separated by a slash (/) and preceded by an optional sign. For example, 3/4, -6/5, and 1/1208203823 are all exact rational numbers. A ratio is reduced immediately to lowest terms when it is read and may in fact reduce to an exact integer.

Inexact real numbers are normally written in either floating-point or scientific notation. Floating-point notation consists of a sequence of numerals followed by a decimal point and another sequence of numerals, all preceded by an optional sign. Scientific notation consists of an optional sign, a sequence of numerals, an optional decimal point followed by a second string of numerals, and an exponent; an exponent is written as the letter e followed by an optional sign and a sequence of numerals. For example, 1.0 and -200.0 are valid inexact integers, and 1.5, 0.034, -10e-10 and 1.5e-5 are valid inexact rational numbers. The exponent is the power of ten by which the number preceding the exponent should be scaled, so that 2e3 is equivalent to 2000.0.

The exactness of a numeric representation may be overridden by preceding the constant by either 0e or 0i. 0e forces the number to be exact, and 0i forces it to be inexact. For example, 1, 0e1, 1/1, 0e1/1, 0e1.0, and 0e1e0 all represent the exact integer 1, and 0i3/10, 0.3, 0i0.3, and 3e-1 all represent the inexact rational 0.3.

Numbers are written by default in base 10, although the special prefixes 0b (binary), 0o (octal), 0d (decimal), and 0x (hexadecimal) can be used to specify base 2, base 8, base 10, or base 16. For radix 16, the letters a through f or A through F serve as the additional numerals required to express digit values 10 through 15. For example, 0b10101 is the binary equivalent of 2110, 0o72 is the octal equivalent of 5810, and 0xC7 is the hexadecimal equivalent of 19910. Numbers written in floating-point and scientific notations are always written in base 10.

If both are present, radix and exactness prefixes may appear in either order.

function: number_is_eq(num1, ...) 
function: number_is_lt(num1, ...) 
function: number_is_gt(num1, ...) 
function: number_is_lteq(num1, ...) 
function: number_is_gteq(num1, ...) 
returns: true if the relation holds, false otherwise.

The predicate number_is_eq returns true if its arguments are equal. The predicate number_is_lt returns true if its arguments are monotonically increasing, i.e., each argument is greater than the preceding ones. The predicate number_is_gt returns true if its arguments are monotonically decreasing, i.e., each argument is less than the preceding ones. The predicate number_is_lteq returns true if its arguments are monotonically nondecreasing, i.e., each argument is not less than the preceding ones. The predicate number_is_lteq returns true if its arguments are monotonically nonincreasing.

slogan> number_is_eq(7, 7); 
=> true
slogan> number_is_lteq(1, 2, 3, 3, 4, 5);
=> true

Fixnums

Fixnums represent exact integers in the fixnum range, which is required to be a closed range [-2w-1, 2w-2 - 1], where w (the fixnum width) is at least 24. The platform-specific value of w may be determined via the function fixnum_width, and the endpoints of the range may be determined via the functions least_fixnum and greatest_fixnum.

The names of arithmetic functions that operate only on fixnums begin with the prefix "fx" to set them apart from their generic counterparts.

Characters

Characters are atomic objects representing letters, digits, special symbols such as $ or -, and certain nongraphic control characters such as space and newline. Characters are written enclosing them in single quotes ('). The written character representation of the letter A, for example, is 'A'. Backslash is used as an escape for special characters. Some common examples are given below:

  • '\n' - newline
  • '\r' - carriage return
  • '\t' - tab
  • '\\' - backslash
  • '\b' - backspace
  • '\a' - alarm
  • '\v' - vertical tab
  • '\e' - escape
  • '\d' - delete

Any Unicode character may be written with the syntax '\xhh', '\uhhhh' or '\Uhhhhhhhh' where n consists of two, four or eight hexadecimal digits representing a valid Unicode scalar value.

This section describes the operations that deal primarily with characters.

Strings

Strings are sequences of characters and are often used as messages, character buffers, or containers for blocks of text. A string is written as a sequence of characters enclosed in double quotes, e.g., "hi there". A double quote may be introduced into a string by preceding it by a backward slash, e.g., "two \"quotes\" within". A backward slash may also be included by preceding it with a backward slash, e.g., "a \\slash". Various special characters can be inserted with other two-character sequences, e.g., \n for newline, \r for carriage return, and \t for tab. Any Unicode character may be inserted with the syntax \xhh, \uhhhh or \Uhhhhhhhh where h consists of two, four or eight hexadecimal digits and represents a valid Unicode scalar value.

Strings are indexed by exact nonnegative integers, and the index of the first element of any string is 0. The highest valid index for a given string is one less than its length.

Symbols

Symbols are used for a variety of purposes as symbolic names in Slogan programs. Strings could be used for most of the same purposes, but an important characteristic of symbols makes comparisons between symbols much more efficient. This characteristic is that two symbols with the same name are identical in the sense of is_eq. The reason is that Slogan makes sure that two symbols with the same sequence of characters are always stored in the same location in memory.

All characters valid in identifiers can be used in symbols. Symbol constants are written by prefixing identifiers with the exclamation mark (!).

function: string_to_symbol(string)
returns: a symbol whose name is string.
slogan> is_eq(!hello, string_to_symbol("hello")); 
=> true
function: symbol_to_string(symbol)
returns: the name of symbol as a string.
