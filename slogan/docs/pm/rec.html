Records

This chapter describes the means by which the programmer may define new data types called records, each distinct from all other types. A record type determines the number and names of the fields each instance of the type has. Records are defined via the record syntactic form:

syntax: record name (field1 = default_value where pre_condition, ...)

Name is the name of the new record and field1 ... are the names of its fields. The where clause, which specifies the pre-conditions that the values assigned to the fields should obey, is optional. The default_value is also optional and defaults to false.

The record statement automatically defines the following functions for the new record type:

function: name(@key field1 = value1, ...) 
function: make_name(value1, value2) 
returns: a new instance of the record named name with its fields set to the given values.

These are the constructor functions for the record named name. The make_name function expects all the fields to be specified in the same order as they were declared by the record statement.

function: is_name(obj) 
returns: true if obj is an instance of the record named name, false otherwise.
function: name_field(r) 
returns: the current value bound to field in the record instance r.
function: name_set_field(r, value) 
returns: void

Binds value to the field named field in the record object r.

An example

This section demonstrates the functions we have discussed so far by defining a simple point record.

slogan> record point (x, y); 
slogan> let p = point(y = 100, x = 20); 
slogan> let p2 = make_point(1, 3.4); 
slogan> point_x(p); 
=> 20
slogan> point_x(p2); 
=> 1
slogan> point_set_y(p2, 5.6); 
slogan> point_y(p2); 
=> 5.6

The point constructor also allows us to leave out fields so that they can take default values:

slogan> p = point(x = 12); 
slogan> point_x(p); 
=> 12
slogan> point_y(p); 
=> false

A coordinate taking the default value of false does not look right. So let us change the record definition a bit:

slogan> record point (x = 0, y = 0);

slogan> p = point(x = 12); 
slogan> point_x(p); 
=> 12
slogan> point_y(p); 
=> 0

Using the where clause, if can restrict the range of values being assigned to the coordinates:

slogan> record point (x = 0 where x >= -80 && x <= 80, y = 0 where y >= 0 && y <= 80); 
slogan> p = point(x = -100, y = 10); 
error: Precondition failed: (and (>= x -80) (<= x 80)). -100
slogan> p = point(x = -10, y = 10); 
slogan> point_set_y(p, -1); 
error: Precondition failed: (and (>= y 0) (<= y 80)). -1
slogan> point_set_y(p, 1);

Keep in mind that the make_point constructor allows us to bypass the pre-conditions. The following code will not raise an error:

slogan> p = make_point(-100, -10);
Pattern matching records

This section shows how to use the match statement to pattern match records through a few simple examples.

slogan> record point (x, y); 
slogan> record three_d_point (x, y, z); 

slogan> function f(p) 
          match (p) 
           point(x, y) -> showln("point ", x, ":", y), 
           three_d_point(x, y, z) -> showln("3d_point ", x, ":", y, ":", z); 

slogan> f(point(x = 10, y = 20)); 
-> point 10:20
slogan> f(three_d_point(x = 10, y = 20, z = 3.4)); 
-> 3d_point 10:20:3.4

The match statement can also have a where clause that can be used to impose more restrictions on how the pattern is matched. For example, if we want to match a point object only if its x is 10, we would rewrite the match statement as follows:

slogan> function f(p) 
          match (p) 
           point(x, y) where x == 10 -> showln("point ", x, ":", y), 
           three_d_point(x, y, z) -> showln("3d_point ", x, ":", y, ":", z); 

slogan> f(point(x = 10, y = 20)); 
-> point 10:20
slogan> f(point(x = 5, y = 20)); 
error: No match found.

The where clause can be used to match any object pattern, not just records:

slogan> match ([1, 2, 3]) 
         [_, b, _] where b >= 10 -> !hi, 
         [_, b, _] where b >= 1 -> !hello; 
=> hello

slogan> match([1, 2, 3]) 
         [_, b, _] where b >= 10 -> !hi, 
         [_, b, _] where b >= 3 -> !hello; 
error: No match found.

slogan> match ([1, 20, 3]) 
         [_, b, _] where b >= 10 -> !hi, 
         [_, b, _] where b >= 1 -> !hello; 
=> hi
