System Interface

This chapter presents the functions and facilities that enable Slogan programs to interact with the host operating system. We cover topics such as accessing the file system, processing command-line arguments and compiling programs to machine code.

Working with Filesystem paths

A path is a string that denotes a file, for example "src/readme.txt".

function: current_directory(@optional new_current_directory) 
returns: the absolute normalized path of the current directory or void.

Calling this function with one argument sets the directory to new-current-directory and returns void.

function: path_expand(path ,@optional origin_directory) 
returns: an expanded path, which is an absolute path when path or origin_directory 
         are absolute paths.
function: path_normalize(path ,@optional allow_relative ,origin_directory) 
returns: the normalized form of path.

All components of the paths path and origin_directory must exist, except possibly the last component of path. A normalized path is a path containing no redundant parts and which is consistent with the current structure of the filesystem. A normalized path of a directory will always end with a path separator (i.e. ‘/’, ‘\’, or ‘:’ depending on the operating system). The optional allow_relative parameter, which defaults to false, indicates if the path returned can be expressed relatively to origin_directory: a false requests an absolute path, the symbol 'shortest requests the shortest of the absolute and relative paths, and any other value requests the relative path. The shortest path is useful for interaction with the user because short relative paths are typically easier to read than long absolute paths.

slogan> path_expand("../f"); 
=> /home/slogan/../f
slogan> path_normalize("../f"); 
=> /home/f
slogan> path_normalize("../f", 'shortest); 
=> ../f

Filesystem Operations

function: create_directory(path ,@optional permissions = 0o777) 
returns: void

This function creates a directory. The argument path must be a string that indicates the location of the directory to create in the filesystem. Permissions must be a 12-bit-exact-integer that controls the UNIX permissions that will be attached to the directory if it is created. The default value of this setting is 0o777.

function: create_link(source_path ,destination_path) 
function: create_symbolic_link(source_path ,destination_path) 
returns: void

These functions create a hard or symbolic link between source_path and destination_path.

function: rename_file(source_path ,destination_path) 
returns: void

This function renames the file source_path to destination_path.

function: copy_file(source_path ,destination_path) 
returns: void

This function copies the file source_path to destination_path.

function: delete_file(path) 
returns: void

This function deletes the file path which must be the path to an existing file.

function: delete_directory(path) 
returns: void

This function deletes the directory path which must be the path to an existing directory.

function: list_directory(path ,@optional ignore_hidden) 
returns: a list of files in the directory path.

The argument ignore_hidden controls whether hidden-files will be returned. Under UNIX and Mac OS X hidden-files are those that start with a period (such as '.', '..', and '.profile'). A setting of false will enumerate all the files. A setting of true will only enumerate the files that are not hidden. A setting of 'dot_and_dot_dot will enumerate all the files except for the '.' and '..' hidden files. The default value of this setting is true.

File Information
function: file_exists(path ,@optional chase) 
returns: true if the file path exists, false otherwise.

When chase is false (the default), symbolic links will not be chased, in other words if path refers to a symbolic link, file_exists will return true whether or not it points to an existing file.

function: file_info(path ,@optional chase) 
returns: a file-information record.

A file-information record is returned that contains the file's type, the device number, the inode number, the mode (permission bits), the number of links, the file's user id, the file's group id, the file's size in bytes, the times of last-access, last-modification and last-change, the attributes, and the creation time.

When chase is false (the default), symbolic links will not be chased, in other words if path refers to a symbolic link the file-info procedure will return information about the link rather than the file it links to.

returns: the various fields of the file-information record fi.

For file_info_type, the returned value is one of the following symbols:

  • 'regular - regular file
  • 'directory
  • 'character_special - character special device
  • 'block_special - block special device
  • 'fifo
  • 'symbolic_link
  • 'socket
  • 'unknown - file is of an unknown type

For file_info_last_access_time, file_info_last_modification_time, file_info_last_change_time and file_info_creation_time the return value is a UNIX timestamp. (seconds elapsed since the UNIX epoch).

Host Information

function: os_name() 
returns: the operating system name as a string, false if the OS name cannot be determined.
function: host_name() 
returns: the machine's host name as a string.
function: host_info(host_name) 
returns: a host-information record.

This function accesses the internet host database to get information about the machine whose name is denoted by the string host_name. The host-information record that is returned will contain the official name of the machine, a list of aliases (alternative names), and a non-empty list of IP addresses for this machine. An exception is raised when host_name does not appear in the database.

returns: the exit status of the shell.

This function calls up the shell to execute command which must be a string.

returns: unspecified

Terminates the current process with the status status which must be an exact integer in the range 0 to 255.

function: command_line()

returns: the value of the environment variable name of the current process.

The function setenv changes the binding of the environment variable name to new_value which must be a string. If new_value is not specified the binding is removed.

Compilation

function: compile(script, @key assemble, exe, ld_options, cc_options, output, exception_handler) 
returns: void

Compiles the file script containing Slogan source code into machine code. If the argument exe is true the file script is compiled to produces a standalone executable. If assemble is true, only a dynamically loadable file of machine instructions is generated. If provided, ld_options must be a string that specifies additional command-line options for the linker. If provided, cc_options must be a string that specifies additional command-line options for the C compiler. The name of the generated file can be customized using output. If provided, exception_handler must be a function that takes a single argument. If compilation raises an exception, this function will be called with the error object as the argument.

function: load(script, @optional force_compile) 
returns: void

Loads the object code file named script. If it is not found looks for script.sn, compiles it into the intermediate language and loads it into the runtime system. As load compiles to the intermediate form, it should be used for loading scripts into the REPL and loading scripts that contain macros. If force_compile is true, the file script.sn is compiled even if a previous version of the intermediate file exists.

function: reload(script) 
returns: void

Reload calls load with force_compile = true.

function: link(script) 
returns: void

Loads the object code file named script. If it not found looks for script.sn, compiles it into machine code and loads it.

function: process_times() 
returns: a three element list containing the cpu time that has been used by the program and 
         the real time that has elapsed since it was started.

The first element of the result list corresponds to "user" time in seconds, the second element corresponds to "system" time in seconds and the third element is the elapsed real time in seconds. On operating systems that can't differentiate user and system time, the system time is zero. On operating systems that can't measure cpu time, the user time is equal to the elapsed real time and the system time is zero.

function: real_time() 
returns: the real time that has elapsed since the program was started.
function: cpu_time() 
returns: the cpu time in seconds that has been used by the program (user time plus system time).
macro: time(expr) 
returns: the result of evaluating the expression expr.

As a side effect it displays a message on the standard output which indicates how long the evaluation took (in real time and cpu time), how much time was spent in the garbage collector, how much memory was allocated during the evaluation and how many minor and major page faults occurred.
