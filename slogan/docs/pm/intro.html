Introduction

Slogan is high-level, general-purpose programming language with support for first-class functions, lexical scoping, multi-tasking, lazy evaluation and continuations. It is designed to be simple and easy to use but powerful enough to write real-world applications.

Slogan Syntax

Slogan programs are made up of keywords, variables, constant data, whitespace, and comments.

Keywords, variables, and symbols are collectively called identifiers. Identifiers may be formed from letters, digits and these special characters: _, $, ?, ~ and @. Identifiers starting with a '?' or a '@' are interpreted specially in certain contexts. We will cover these special contexts in later chapters. Some characters are reserved by the language for special purposes, mostly to be used as operators. These characters that cannot be used in identifiers are: +, -, /, *, <, >, =, !, #, ., (, ), {, }, [, ], ,, :, %, ; & and |.

Identifiers are case-sensitive. For example, abcd, Abcd and ABcD all refer to different identifiers. There is no inherent limit to the length of identifiers.

Constant data refers to values that has a literal expression in the language. These values are one of the following types: boolean, number, character, string, array, list, symbol and function.

A boolean values are represented by the identifiers true and false. Numbers may be written as integers, e.g., -123, as ratios, e.g., 1/2, in floating-point or scientific notation, e.g., 1.3 or 1e23. Numbers is different bases can be written with a radix prefix. For example, a hexadecimal number is written as 0xFF, an octal number as 0o76 and a binary number as 0b0011. Case is not important in the syntax of a number. Strings are enclosed in double quotation marks, for e.g., "hello, world". Characters are enclosed in single quotation marks as in 'a' and 'b'. Backslash is used to introduce a special character. For e.g., the newline constant is '\n' and a double quote literal in a string is '\"'. Symbols are names prefixed by the ! operator.

A list constant consists of comma-separated values enclosed in brackets: ["this", "is", "a", "list"]. An array is similar to a list but the opening bracket is prefixed by the hash sign: #["this", "is", "an", "array"].

Function constants are introduced by the function or fn keywords. For e.g., this is a function literal that takes a number as argument and finds its square: fn (x) x * x.

Keywords, variables and constant data combine to make up Slogan expressions and statements. An expression or a statement can span multiple lines and must be terminated by a semi-colon (;). An expression always results in a value.

Comments explaining a particular Slogan expression can be placed in the program. Single line comments starts with a // and multi-line comments are enclosed in /* and */.
