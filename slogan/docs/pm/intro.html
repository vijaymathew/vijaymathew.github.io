Introduction

Slogan is a high-level, general-purpose programming language with support for first-class functions, lexical scoping, multi-tasking, lazy evaluation and continuations. It is designed to be easy to learn, fun to use and powerful enough to write real-world applications.

Slogan Syntax

Slogan programs are made up of keywords, variables, constant data, whitespace, and comments.

Keywords, variables, and symbols are collectively called identifiers. Identifiers may be formed from letters, digits and these special characters: _, $, %, and @. (Identifiers beginning with a '?' are treated specially in certain contexts. We will cover these special contexts in later sections.) Some characters are reserved by the language for special purposes, mostly to be used as operators. The characters that cannot normally be used in identifiers are: +, -, /, *, <, >, =, !, #, ., (, ), {, }, [, ], ,, :, ; & and |.

Identifiers are case-sensitive. For example, abcd, Abcd and ABcD all refer to different identifiers. There is no inherent limit to the length of identifiers.

A number of identifiers are reserved by Slogan for syntactic definition. These are known as reserved words or keywords. They cannot be used as names for variables. The reserved words in Slogan are: case, catch, declare, else, false, finally, ^, function, if, and, or, let, letrec, letseq, macro, match, method, module, record, true, try, trycc, where, when (since v0.2), yield (since v0.3) and assert (since v0.5).

Constant data refers to values that has a literal expression in the language. These values are one of the following types: boolean, number, character, string, array, list, symbol and function.

Boolean values are represented by the identifiers true and false. Numbers may be written as integers, e.g., -123, as ratios, e.g., 1/2, in floating-point or scientific notation, e.g., 1.3 or 1e23. Numbers is different bases can be written with a radix prefix. For example, a hexadecimal number is written as 0xFF, an octal number as 0o76 and a binary number as 0b0011. Case is not important in the syntax of a number. Strings are enclosed in double quotation marks, for e.g., "hello, world". Characters are enclosed in single quotation marks as in 'a' and 'b'. Backslash is used to introduce a special character. For e.g., the newline constant is '\n' and a double quote literal in a string is '\"'. Symbols are names prefixed by the ! operator.

A list literal consists of comma-separated values enclosed in brackets: ["this", "is", "a", "list"]. An array is similar to a list but the opening bracket is prefixed by the hash sign: #["this", "is", "an", "array"].

Functions are introduced by the function or ^ keywords. For e.g., this is a function literal that take a number as argument and find its square: ^(x) x * x.

Keywords, variables and constant data combine to make up Slogan expressions and statements. An expression or a statement can span multiple lines and must be terminated by a semi-colon (;). An expression always results in a value.

Comments explaining a particular expression can be placed in the program. Single line comments starts with a // and multi-line comments are enclosed in /* and */.

Evaluating simple expressions

Slogan comes with a simple interactive programming environment that eases program development and experimentation. This program that reads each expression you type at the keyboard, evaluates it, and prints its value is known as the read-evaluate-print-loop or the REPL.

With the REPL, you can type an expression at the keyboard and see its value immediately. You can type in functions or even an entire program consisting of a set of function definitions and test them without leaving the system. But it is more convenient to type longer programs into a text file, load it into the REPL and test it interactively. A file is loaded with the function load, which is passed the name of the file as a string. By preparing your program in a file you have the chance to correct errors without retyping the program, and you can retain a copy for later use.

The examples in this document follow a regular format. An expression you might type from your keyboard is given first, preceded by the REPL prompt "slogan>". The value of the expression is given after "=>", to be read as "evaluates to." Output produced as the result of a side-effect is identified by "->". Let us see two examples. The first is an expression that evaluates to the value 7 and the next is a statement that prints the string "hello!" as a side-effect:

slogan> 3 + 4; 
=> 7
slogan> showln("hello!"); 
-> hello!

Expressions may span more than one line. Slogan know it has a complete expression to evaluate when it sees a semicolon (;). Let's define a function that spans multiple lines:

slogan> function reciprocal(n) 
          if (n == 0) "oops!" 
          else 1 / n;

The REPL prompt will change from "slogan>" to ">" for each new line in the expression. The prompt will return to "slogan>" once Slogan has read and evaluated the entire expression. I have omitted the ">" prompt from the examples to keep the code easier to read.

The function we just defined, reciprocal, computes the quantity 1/n for any number n =/ 0. For n == 0, reciprocal returns the string "oops!". Let us try using this function:

slogan> reciprocal(10); 
=> 1/10
slogan> reciprocal(1/10); 
=> 10
slogan> reciprocal(0); 
=> "oops!"

Let's turn to a discussion of how Slogan evaluates the expressions you type. The rule for evaluating constant objects such as strings and numbers is simple: the object itself is the value. For example, if you type the string "hello, world" at the REPL, the string is printed back to you. What are the rules for evaluating a function call of the form f(arg1, ..., argn)?. Here, f is an expression representing a Slogan function, and arg1, ..., argn are expressions representing its arguments. The rules for finding the value of this expression is:

  1. Find the value of f.
  2. Find the values of arg1 ... argn.
  3. Apply the value of f to the values of arg1 ... argn.

For example, consider the function call reciprocal(10). The value of reciprocal is the reciprocal function we defined. The value of 10 is the number 10. Applying the reciprocal function to 10 yields 1/10, so our value is the object 1/10.

Let expressions

Suppose you want to assign a name to a value that you use in expressions. For example, we might like the name x to have the value 2 when we evaluate 3 + x. Such names or identifiers bound to a value is known as a variable. We can use a let expression to bind identifiers to values:

slogan> let (x = 2) 3 + x; 
=> 5
slogan> let (y = 3) 2 + y; 
=> 5
slogan> let (x = 2, y = 3) x + y; 
=> 5

The let expression includes a list of variable assignments, followed by an expression known as the body. The general form of a let expression is 

let (var = expr, ...) body

We refer to variables bound by let as let-bound variables. The variables bound by let are visible only within the body of the let.

It is possible to nest let expressions:

slogan> let (a = 4, b = -3) 
         let (a_squared = a * a, b_squared = b * b) 
          a_squared + b_squared;
=> 25

When nested let expressions bind the same variable, only the binding created by the inner let is visible within its body:

slogan> let (x = 1) 
          let (x = x + 1) 
           x + x; 
=> 4

The inner binding for x is said to shadow the outer binding. A let-bound variable is visible everywhere within the body of its let expression except where it is shadowed. The region where a variable binding is visible is called its scope. The scope of the first x is the body of the outer let expression minus the body of the inner let expression, where it is shadowed by the second x. This form of scoping is referred to as lexical scoping, since the scope of each binding can be determined by a straightforward textual analysis of the program.

Shadowing may be avoided by choosing different names for variables. Although choosing different names can sometimes prevent confusion, it can help prevent the accidental use of an "old" value. For example, in the previous example, it would be impossible for us to mistakenly refer to the outer x within the body of the inner let.

A letseq expression is similar to let except that the expressions expr, ... are evaluated in sequence from left to right, and each of these expressions is within the scope of the variables to the left. Use letseq when there is a linear dependency among the values or when the order of evaluation is important.

slogan> letseq (x = 5 * 5, y = x - 4 * 4) 
         sqrt(y); 
=> 3

Any letseq expression may be transformed into a set of nested let expressions as shown in the following program:

slogan> let (x = 5 * 5) 
         let (y = x - 4 * 4) 
          sqrt(y); 
=> 3

Function expressions

The general form of a function expression is

function (var, ...) body

or

^(var, ...) body

The variables var, ... are the formal parameters of the function, and the expression body is its body. A function is an object just like a number or a string but it does not have a meaningful printed representation. Slogan will print <function> for all function objects.

The most common operation to perform on a function is to apply it to one or more values:

slogan> (function (x) x + x) (3 * 4); 
=> 24

The function is the value of function (x) x + x, and the only argument is the value of 3 * 4, or 12. The argument values, or actual parameters, are bound to the formal parameters within the body of the function expression in the same way as let-bound variables are bound to their values. In this case, x is bound to 12, and the value of x + x is 24. Thus, the result of applying the function to the value 12 is 24.

Because functions are objects, we can establish a function as the value of a variable and use the function more than once:

slogan> let (double = ^(x) x + x) 
         [double(3 * 4), 
          double(99 / 11), 
          double(2 - 7)]; 
=> [24, 18, -10]

Here, we bind the identifier double to a function and use this function to double three different values.

Conditional expressions

An if expression allows us to perform a task conditionally. The if expression has the following form: 

if (test) consequent else alternative

The consequent expression is evaluated if test is true and the alternative expression is evaluated if test is false. (All objects except the symbol false is considered to be true). The alternative expression is optional. If test is false and alternative is not provided, if will evaluate to false.

(since v0.2) The when expression is similar to if without the alternative part:

when (test) consequent

The comparison operators ==, <>, <, >, <= and >= are used for comparing numbers. They return true if the comparison succeeds and return false otherwise. The function not is used to reverse a boolean value.

slogan> if (10 > 20) "no";
=> false      
slogan> if (10 > 20) "no" else "yes"; 
=> "yes"
slogan> if (not(10 > 20)) "no" else "yes"; 
=> "no"
slogan> when (not(10 > 20)) "no"; 
=> "no"
slogan> when (10 > 20) "yes"; 
=> false

The logical operators && (and) and || (or) are typically used with boolean values. When they are, they return a boolean value. However, if these operators are used with non-boolean values, they may return a non-boolean value:

slogan> true && false;
=> false
slogan> true && true;
=> true
slogan> true || false;
=> true
slogan> false || false;
=> false
slogan> false || 10;
=> 10
slogan> 1 && "hi";
=> "hi"

The macros and and or can be used in place of the logical operators. They have the advantage that they can take an arbitrary number of arguments:

slogan> and(true, 90, 80, 100);
=> 100
slogan> and(true, 90, 80, false, 100);
=> false
slogan> or(true, 90, 80, false, 100);
=> 100

Though macros are called using the same syntax as functions, macro arguments are evaluated only at the point of their use in the macro body. This is made clear in following example:

slogan> and(true, 1/0); 
-> error: Divide by zero
slogan> and(false, 1/0); 
-> false

If and was a normal function, both call would have raised the "Divide by zero" error. As it is a macro, and will not evaluate the expression 1/0 during the second call as it has already decided that it can only evaluate to false. In a later chapter we will see how to define macros. We will also learn about functions that has the ability to evaluate their arguments only when needed.

Top-level definitions

Top-level variable bindings that are visible to all expressions that follow it are introduced by the let statement, which has the form:

let var = expr;
let var1, var2, ...varN = expr1, expr2, ...exprN;

where the identifier var is bound to the value of the expression expr. Let us see some examples:

slogan> let x = 10;
slogan> let y = 20;
slogan> x + y; 
=> 30

slogan> let square = ^(x) x * x; 
slogan> square(10); 
=> 100

There is a shorter syntax for defining top-level functions. The following definition of square is the same as the previous one:

slogan> function square(x) x * x;

Let statements can also appear within a let or function expression, at the top of its body. These definitions will not be visible outside the body. As the let or function body is a single expression, we will have to group the definitions and the body into a single expression by using a code block. A code block is a sequence of statements and expressions enclosed in { and }. The value of a code block is the value of its last expression. In the following program the top-level f is shadowed by an internal definition introduced by the body of the let expression:

slogan> function f(x) x * x;
slogan> let (y = 3) 
        { let f = ^(x) y + x; 
          f(4) }; 
=> 7
slogan> f(4);
=> 16

A code block is valid anywhere another expression is valid. The following program shows two functions defined internally to a code block in the top-level. (This is an example of mutual recursion which we will study in more detail in the next chapter.)

slogan> { function is_even(x) or(x == 0, is_odd(x - 1)); 
          function is_odd(x) and(not(x == 0), is_even(x - 1)); 
          [is_even(20), is_odd(20)] }; 
=> [true, false]
Assignments

Although many programs can be written without them, assignments to top-level variables or let-bound and function-bound variables are sometimes useful. Assignments do not create new bindings, as with let or function, but rather change the values of existing bindings. Assignments are performed with the = operator:

slogan> let a = 100; 
slogan> a * 10; 
=> 1000
slogan> a = 20;
slogan> a * 10;
=> 200

Backquoting Operators

The arithmetic, comparison and logical operators are internally mapped to functions. The function object associated with an operator can be accessed by backquoting the operator:

slogan> `+`(1, 2, 3); 
=> 6
slogan> `&&`(`<`(1, 2), `>=`(10, 10)); 
=> true

Backquoting can also create identifiers with reserved characters and whitespaces:

slogan> let `two + two` = 2 + 2; 
slogan> `two + two` * 100; 
=> 400

You should use backquoted definitions with caution. For instance, re-defining `+` will break Slogan itself as this function is used in its implementation.

The comma separator

The comma operator (,) is used to separate values in a sequence, both visually and semantically. In most scenarios where comma is used as a separator, you can omit it and use whitespace instead. For example, all the list and array literals shown below are treated the same by the Slogan interpreter:

slogan> [1 ,2 , 3];
=> [1, 2, 3]
slogan> [1 2 3];
=> [1, 2, 3]
slogan> #[1, 2, 3];
=> #[1, 2, 3]
slogan> #[1 2 3];
=> #[1, 2, 3]

The comma separator is optional in function declaration and calls as well:

slogan> function f(a b) a + b * 100; 
slogan> f(10 20); 
=> 2010

Though omitting comma can be convenient in some situations, it is recommended that you always type them in. This helps in improving the readability and consistency of the code and prevent subtle bugs such as the one shown below:

slogan> [1 -10 20]; 
=> [-9, 20]

We wanted to create the list [1, -10, 20] but as there was no separator between 1 and -10 they were interpreted as being part of the single expression 1 - 10 and was evaluated to produce -9 in the final list. This bug could have been avoided if we used comma separator explicitly:

slogan> [1, -10, 20];
=> [1, -10, 20]

While dealing with lists of other datatypes like symbols, you may decide that it is more visually appealing to omit commas:

slogan> %[this is a list of seven symbols]; 
=> [this, is, a, list, of, seven, symbols]
slogan> length(%[this is a list of seven symbols]); 
=> 7

Commas in the printed output can be toggled on/off by calling the show_comma_separator function:

slogan> show_comma_separator(false); 
slogan> %[this is a list of seven symbols]; 
=> [this is a list of seven symbols]
