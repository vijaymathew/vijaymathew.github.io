Exceptions

Exceptions provide the means for system and user code to signal, detect, and recover from errors that occur when a program is run.

function: current_exception_handler()
returns: the current exception handler function.
function: set_current_exception_handler(f)
returns: void

Sets the current exception handler function to f which takes a single argument that represents the exception currently being handled. The current exception handler function is not active in the REPL. To test these functions you should compile a script. For example, compile the following scripts, run the executables and see how changing the current exception handler affect the way error is reported:

// test1.sn
1 / 0; 
// test2.sn
set_current_exception_handler(fn (ex) showln("ERROR CAUGHT!!", ex)); 
1 / 0;

(To compile the scripts to executables, see the reference on the compile function in the System Interface chapter.)

syntax: try expr1 
        catch (exception) expr2 
        finally expr3 
syntax: trycc expr1 
        catch (exception) expr2 
        finally expr3
returns: the value of evaluating the expression expr1

If expr1 raises an exception, this is passed to the catch block as the object exception. The code in expr2 in the catch block can then deal with this object in a suitable manner. The finally clause is optional. If it is provided, expr3 will be evaluated even in the case of an exception.

Trycc allows the continuation of the raise function to be invoked if expr2 returns normally. It also has the disadvantage that an exception raised in expr2 may lead to an infinite loop, if not handled within expr2 itself. In all normal circumstances, try should be preferred over trycc.

function: raise(obj)
returns: void

This function tail-calls the current exception-handler with obj as the sole argument. If the exception-handler returns, the continuation of the call to raise is invoked. (If called within a try or trycc statement the current exception-handler is the expression in the catch clause.)

slogan> try 100 + raise(!oops) 
        catch (e) { showln(e); 2 }; 
-> oops
=> 2
slogan> trycc 100 + raise(!oops) 
        catch (e) { showln(e); 2 }; 
-> oops
=> 102
function: abort(obj)
returns: void

The function abort calls the current exception-handler with obj as the sole argument. If the exception-handler returns, the function abort will be tail-called with a noncontinuable-exception object, whose reason field is obj, as sole argument.

slogan> callcc(fn (k) 
                try 1 + abort(!hello) 
                catch (e) 
                 { showln(e); 
                   if (is_noncontinuable_exception(e)) 
                    k(noncontinuable_exception_reason(e))
                   else 100 }); 
-> hello
=> 100

slogan> callcc(fn (k) 
                trycc 1 + abort(!hello) 
                catch (e) 
                 { showln(e); 
                   if (is_noncontinuable_exception(e)) 
                    k(noncontinuable_exception_reason(e))
                   else 100 });
-> hello
-> #<noncontinuable-exception #3>
-> hello
function: is_noncontinuable_exception(obj)
returns: true if obj is a noncontinuable-exception object, false otherwise.
function: noncontinuable_exception_reason(e)
returns: the reason associated with noncontinuable-exception object e.
function: show_exception(e, @optional port)
returns: void

Prints the human-readable representation of the exception object e to the textual output port port.

Error-Exception Objects

function: error(message, obj1, ...)
returns: unspecified.

This function raises an error-exception object whose message field is message and arguments field is the list of values obj1 ....

function: is_error(obj)
returns: true if obj is a error-exception object, false otherwise.
function: error_message(e)
function: error_args(e)
returns: the message or arguments associated with the error-exception object e.
