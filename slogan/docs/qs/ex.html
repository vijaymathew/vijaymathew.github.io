Exceptions

The raise function is used to report an error condition. The try-catch expression can handle such exceptional events.

slogan> function divide(a, b) 
         if (is_zero(b)) raise("Cannot divide by zero.") 
         else a / b; 

slogan> try divide(10, 2) 
        catch (ex) showln(ex); 
=> 5
slogan> try divide(10, 0) 
        catch (ex) showln(ex); 
-> Cannot divide by zero.

Try has an optional finally clause that is guaranteed to execute before try returns. This is useful for running clean-up code that must be executed under all circumstances:

slogan> try raise(100) 
        catch (ex) showln(ex) 
        finally { showln("returning a default result..."); 
                  0 }; 
-> 100
-> returning a default result...
=> 0

More information on raising and handling exceptions can be found in the manual.
