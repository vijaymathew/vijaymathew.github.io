Functions, Generics, Methods

The following function add 2 to its argument:

slogan> function add_2(x) x + 2;

Functions are first-class objects like numbers, strings and lists. The short ^ (also spelt "function"!) keyword can be used to write function literals:

slogan> let add_2 = ^(x) x + 2;

Irrespective of how you define a function, there is just one way to call it:

slogan> add_2(4); 
=> 6
slogan> add_2(12.32); 
=> 14.32

Some or all parameters of a function can be given default values:

slogan> function add_n(x, n= 2) x + n; 
slogan> add_n(4); 
=> 6
slogan> add_n(4, 10); 
=> 14

Parameters can be "annotated" to make them optional:

slogan> function tax(amount, @optional rate) 
         if (rate) amount * rate 
         else amount * 0.025; 

slogan> tax(1000.45); 
=> 25.011250000000004
slogan> tax(1000.45, 0.001); 
=> 1.00045

Keyword parameters allow arguments to be named while calling the function:

slogan> function tax(@key amount, rate) amount * rate; 

slogan> tax(rate = 0.001, amount = 1000.45); 
=> 1.00045

The @rest annotation packs an arbitrary number of arguments into a list and gives that to the function:

slogan> function sum(a, b, @rest nums) 
         let loop(s = a + b, nums = nums) 
          if (is_empty(nums)) s 
          else loop(s + head(nums), tail(nums)); 

slogan> sum(1, 2); 
=> 3
slogan> sum(1, 2, 3, 4, 5); 
=> 15

Functions can recursively call themselves. Here is a definition of Fibonacci numbers:

slogan> function fib(n) 
         match (n) 
          0 -> 0
          | 1 -> 1
          | else -> fib(n - 1) + fib(n - 2); 

slogan> fib(10); 
=> 55

Recursion at tail-positions are optimized by Slogan so that they don't grow the call-stack. The following program shows a properly tail-recursive function for finding the number of elements in a list:

slogan> function len(xs) 
         letrec (loop = ^(xs, c) 
                         if (is_empty(xs)) c 
                         else loop(tail(xs), c + 1)) 
          loop(xs, 0); 

slogan> len([1 2 3 4 5]); 
=> 5

Tail-call optimization renders special forms for iteration like the for and while looping constructs redundant.

As functions are first-class objects, they can be passed as arguments to other functions. Map and fold are examples of built-in functions that accept functional-arguments:

slogan> map(^(x) x * x, [1 2 3 4 5]); 
=> [1, 4, 9, 16, 25]
slogan> fold(`+`, 0, [1 2 3 4 5]); 
=> 15

It is easy to define functions with function parameters:

slogan> function call_fn(f, a, b) f(a, b); 

slogan> call_fn(`+`, 10, 20); 
=> 30

Here is another example. The function iter defined below calls a function f iteratively n number of times:

slogan> function iter(n, f, x) 
         if (n == 0) x 
         else f(iter(n - 1, f, x)); 

slogan> iter(4, ^(x) x * x, 2); 
=> 65536

We can re-define the Fibonacci function using iter:

slogan> function fib(n) 
         head(iter(n, ^(p) let (x = head(p), y = tail(p)) x + y : x, 0 : 1));

This is more efficient than the earlier one because it computes each Fibonacci number only once.

Functions can return other functions as results. As an example, consider the function compose. It takes an arbitrary number of functions as arguments and produces a function that call these functions in sequence. Each function receives the result of the previous function as argument. The functions are applied from right to left.

slogan> function square(x) x * x; 
slogan> let square_twice = compose(square, square); 

slogan> square_twice(3); 
=> 81

Another interesting function is partial which returns a partially applied version of a function:

slogan> let add_3 = partial(`+`, 3); 
slogan> add_3(5); 
=> 8

Methods are similar to functions but allows us to overload the same function for different types of arguments. For example, let us extend the built-in + (addition) function so that it can also be used to concatenate strings. (As we have already seen in some of the previous examples, operator symbols like + must be escaped with tick-quotes (`) while using them as function names):

slogan> method `+`(s1:string, s2:string) string_append(s1, s2); 

slogan> "hello, " + "world"; 
=> hello, world
slogan> 2 + 3; 
=> 5

Methods are specialized instances of generics: 

slogan> declare generic len (obj); 
slogan> method len(obj:list) length(obj); 
slogan> method len(obj:string) string_length(obj); 

slogan> len([1 2 3 4]);
=> 4
slogan> len("hello"); 
=> 5
slogan> len(!hello); 
error: Method not defined.
