Functions, Generics, Methods

The following function adds 2 to its argument:

function add_2(x) x + 2;

Functions are first-class objects like numbers, strings and lists. The short ^ (also spelt "function") keyword can be used to write function literals:

let add_2 = ^(x) x + 2;

Irrespective of how you define a function, there is just one way to call it:

add_2(4); 
//=> 6
add_2(12.32); 
//=> 14.32

Some or all parameters of a function can be given default values:

function add_n(x, n = 2) x + n; 
add_n(4); 
//=> 6
add_n(4, 10); 
//=> 14

Parameters are "annotated" to make them optional:

function tax(amount, @optional rate) 
  if (rate) amount * rate 
  else amount * 0.025; 

tax(1000.45); 
//=> 25.011250000000004
tax(1000.45, 0.001); 
//=> 1.00045

Keyword parameters allow arguments to be named:

function tax(@key amount, rate) amount * rate; 

tax(rate = 0.001, amount = 1000.45); 
//=> 1.00045

The @rest annotation packs an arbitrary number of arguments into a list and gives that to the function:

function sum(a, b, @rest nums) 
  let loop(s = a + b, nums = nums) 
   if (is_empty(nums)) s 
   else loop(s + head(nums), tail(nums)); 

sum(1, 2); 
//=> 3
sum(1, 2, 3, 4, 5); 
//=> 15

Functions can recursively call themselves. Here is a definition of Fibonacci numbers:

function fib(n) 
| 0 -> 0
| 1 -> 1
| else -> fib(n - 1) + fib(n - 2); 

fib(10); 
//=> 55

Recursion at tail-positions are optimized, so they don't grow the call-stack. The following program shows a properly tail-recursive function for finding the number of elements in a list:

function len(xs) 
  letrec (loop = ^(xs, c) 
                   if (is_empty(xs)) c 
                   else loop(tail(xs), c + 1)) 
    loop(xs, 0); 

len([1, 2, 3, 4, 5]); 
//=> 5

Tail-call optimization renders special forms for iteration like the for and while loops redundant.

As functions are first-class objects, they can be passed as arguments to other functions. Map and fold are examples of built-in functions that accept functional-arguments:

map(^(x) x * x, [1, 2, 3, 4, 5]); 
//=> [1, 4, 9, 16, 25]
fold(`+`, 0, [1, 2, 3, 4, 5]); 
=> 15

It is easy to define functions with function parameters. Here is an example. The function iter defined below calls a function f iteratively n number of times:

function iter(n, f, x) 
  if (n == 0) x 
  else f(iter(n - 1, f, x)); 

iter(4, ^(x) x * x, 2); 
//=> 65536

We can re-define the Fibonacci function using iter:

function fib(n) 
  head(iter(n, ^(p) let (x = head(p), y = tail(p)) x + y : x, 0 : 1));

This is more efficient than the earlier one because it computes each Fibonacci number only once.

Functions can return other functions as results. As an example, consider the function compose. It takes an arbitrary number of functions as arguments and produces a function that call these functions in sequence. Each function receives the result of the previous function as argument. The functions are applied from right to left.

function square(x) x * x; 
let square_twice = compose(square, square); 

square_twice(3); 
//=> 81

Another interesting function is partial which returns a partially applied version of a function:

let add_3 = partial(`+`, 3); 
add_3(5); 
//=> 8

Methods are similar to functions but allows us to overload the same function for different types of arguments. For example, let us extend the built-in + (addition) function so it can also be used to concatenate strings. (Operator symbols like + must be escaped with tick-quotes (`) while using them as function names):

method `+`(s1:string, s2:string) string_append(s1, s2); 

"hello, " + "world"; 
//=> hello, world
2 + 3; 
//=> 5

Methods are specialized instances of "generics": 

declare generic len (obj); 
method len(obj:list) length(obj); 
method len(obj:string) string_length(obj); 

len([1, 2, 3, 4]);
//=> 4
len("hello"); 
//=> 5
len('hello);
//-> error: method_not_defined
