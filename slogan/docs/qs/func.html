Functions, Generics, Methods

The following function adds 2 to its argument:

slogan> function add_2(x) x + 2;

You can also use the shorter fn keyword instead of function:

slogan> fn add_2(x) x + 2;

Functions are first-class objects like numbers, strings and lists. That means we can also define a function like this:

slogan> define add_2 = fn(x) x + 2;

Irrespective of how you define a function, there is just one way to call it:

slogan> add_2(4); 
=> 6
slogan> add_2(12.32); 
=> 14.32

Some or all parameters of a function can be given default values:

slogan> function add_n(x, n= 2) x + n; 
slogan> add_n(4); 
=> 6
slogan> add_n(4, 10); 
=> 14

Parameters can be "annotated" to make them optional:

slogan> function tax(amount, @optional rate) 
         if (rate) amount * rate 
         else amount * 0.025; 

slogan> tax(1000.45); 
=> 25.011250000000004
slogan> tax(1000.45, 0.001); 
=> 1.00045

Keyword parameters allows arguments to be named while calling the function:

slogan> function tax(@key amount, rate) amount * rate; 

slogan> tax(rate = 0.001, amount = 1000.45); 
=> 1.00045

The @rest annotation packs an arbitrary number of arguments into a list and gives that to the function:

slogan> function sum(a, b, @rest nums) 
         let loop(s = a + b, nums = nums) 
          if (is_empty(nums)) s 
          else loop(s + head(nums), tail(nums)); 

slogan> sum(1, 2); 
=> 3
slogan> sum(1, 2, 3, 4, 5); 
=> 15

In the sum function we just defined, the first two arguments are always required. There is a shortcut to define functions where all arguments are optional:

slogan> function sum nums 
         let loop(s = 0, nums = nums) 
          if (is_empty(nums)) s 
          else loop(s + head(nums), tail(nums)); 

slogan> sum(); 
=> 0
slogan> sum(1, 2, 3, 4, 5); 
=> 15

Methods are similar to functions but allows us to overload the same function for different types of arguments. For example, let us extend the built-in + (addition) function so that it can also be used to concatenate strings. (Operator symbols like + must be escaped with tick-quotes (`) while using them as function names):

slogan> method `+` (string, string) (s1, s2) string_append(s1, s2); 

slogan> "hello, " + "world"; 
=> hello, world
slogan> 2 + 3; 
=> 5

Methods are specialized instances of generics: 

slogan> declare generic len (obj); 
slogan> method len (list) (obj) length(obj); 
slogan> method len (string) (obj) string_length(obj); 

slogan> len([1 2 3 4]);
=> 4
slogan> len("hello"); 
=> 5
slogan> len(!hello); 
error: Method not defined.
