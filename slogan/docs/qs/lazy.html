Laziness

The delay operator (~) converts an expression into a "promise" that can be later "forced" to produce a value:

let p = ~{ showln("evaluating 1 + 2 ..."); 1 + 2 }; 
force(p); 
//-> evaluating 1 + 2 ...
//=> 3
force(p); 
//=> 3

Argument expressions can be wrapped in function literals to delay its evaluation until its value is actually required by the function. Such "lazy" arguments are useful in defining control structures that conditionally evaluate their arguments:

function unless(condition, body) when (not(condition)) body(); 

unless(1 < 2, ^showln("ok")); 
unless(1 == 2, ^showln("ok")); 
//-> ok

(As you can see in the above example, the parameter list is optional for function literals that take no arguments.)

A pair can also be lazy, evaluating its tail only when required. The following program shows how to create an infinite sequence of integers using lazy pairs:

function ints(n) n:~ints(n + 1); 

let n = ints(10); 
first(n); 
//=> 10
second(n); 
//=> 11
nth(34, n); 
//=> 44
