Laziness

An expression can be lazily evaluated with a call to delay. Delay returns a "promise" to execute the expression at a later point in time. The function force is used to evaluate a delayed expression. Force remembers the value of the expression and do not evaluate it again for subsequent calls.

slogan> define p = delay({ showln("evaluating 1 + 2 ..."); 1 + 2 }); 
slogan> force(p); 
-> evaluating 1 + 2 ...
=> 3
slogan> force(p); 
=> 3

Evaluation of a function's arguments can also be delayed until its value is actually required by the function. This is achieved by defining a function to be lazy. Lazy functions are useful for defining control structures that conditionally evaluate their arguments:

slogan> function lazy unless(condition, body) 
         if (not(condition)) body; 

slogan> unless(1 < 2, showln("ok")); 
slogan> unless(1 == 2, showln("ok")); 
-> ok

A pair can also be lazy, evaluating its tail only when required. The following program shows how to create an infinite sequence of integers using lazy pairs:

slogan> function ints(n) 
            n :: ints(n + 1); 

slogan> define n = ints(10); 
slogan> first(n); 
=> 10
slogan> second(n); 
=> 11
slogan> nth(34, n); 
=> 44
