Scope, Control, Loops

The let expression introduces local variables:

slogan> let (a = 34 , b = 20) a + b; 
=> 54

Variable bindings in let will shadow global bindings with the same name:

slogan> define a = 10; 
slogan> let (a = 34, b = 20) a + b; 
=> 54

If a binding expression needs to reference the value of a previous binding, letseq should be used:

slogan> letseq (a = 34, b = 20 * a) a + b; 
=> 714

letrec must be used to define recursive local functions:

slogan> letrec (a = function(x) if (is_zero(x)) b(x) else a(x - 1), 
                b = function(y) if (is_zero(y)) !ok else !blah) 
          a(10); 
=> ok

Code-blocks delimited by curly-braces can also be used to limit the scope of variables:

slogan> { define a = 10; 
          define b = 20; 
          a + b }; 
=> 30

The expressions in a code-block are executed sequentially and the value of the last expression is returned.

A let expression can be supplied a "name". This name can be called to repeatedly execute the let body.

slogan> let loop (i = 0) 
         if (i < 5) 
          { showln(i); 
            loop(i + 1) }; 
-> 0
   1
   2
   3
   4

The if expression can be used to execute code conditionally. If has three parts - a condition, a consequent and an alternative. If condition is not false, consequent is evaluated otherwise alternative is evaluated. The alternative part is optional and defaults to a special value called void which have no printed representation.

slogan> if (1 < 2) !ok; 
=> ok
slogan> if (1 > 2) !ok; 
slogan> if (1 > 2) !ok else !blah; 
=> blah

The case expression is used to compare the value of an expression with a number of clauses. The consequent part of the first matching clause is executed. If none of the clauses match, case will return void. Optionally an else part can be specified which will act as the default.

slogan> case (1 + 2) 
         1 -> !one, 
         2 -> !two, 
         3 -> !three; 
=> three
slogan> case (1 - 2) 
         1 -> !one, 
         2 -> !two, 
         3 -> !three, 
         else -> !none; 
=> none

The match expression is similar to case but allow pattern matching and variable bindings using complex clauses:

slogan> match ([1 2 3]) 
         [1 a b] -> a + b, 
         [2 a b] -> a * b; 
=> 5
slogan> match ([2 2 3]) 
         [1 a b] -> a + b, 
         [2 a b] -> a * b; 
=> 6
slogan> match ( [3 2 3]) 
         [1 a b] -> a + b, 
         [2 a b] -> a * b, 
         else ->!none;
=> none

slogan> record point(x, y); 
slogan> record three_d_point(x, y, z); 

slogan> function add(p) 
         match (p) 
          point(x, y) -> x + y, 
          three_d_point(x, y, z) -> x + y + z; 

slogan> add(point(x = 20, y = 13.43)); 
=> 33.43
slogan> add(three_d_point(x = 20, y = 13.43, z = 32.32)); 
=> 65.75
