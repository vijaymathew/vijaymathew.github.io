FFI

FFI or the Foreign Function Interface is a facility that allow Slogan programs to call functions in shared libraries, which are usually written in the C programming language.

The following C program defines two functions that can be called from Slogan. The first function - add_i - expects two integers as arguments and return their sum. The second function - say_hello - consumes and returns a character string:

// demo_lib.c

#include <stdio.h>

int add_i(int a, int b)
{
  return a + b;
}

static char *message = "hi from C";
          
char *say_hello(char *s)
{
  printf("Slogan says: %s\n", s);
  return message;
}

The following command shows how to call the GNU C Compiler to convert this code into a shared library. 

gcc -Wall -shared -fPIC -o demo_lib.so demo_lib.c

On Mac OS X, the build-command will be:

gcc -dynamiclib -Wl,-undefined -Wl,dynamic_lookup -o demo_lib.so demo_lib.c

The declare ffi statement is used to load the library into slogan and define its function prototypes:

slogan> declare ffi "demo_lib.so" [int add_i [int, int],
                                   charstring say_hello [charstring]];

Now we can call these functions from Slogan:

slogan> add_i(10, 20); 
=> 30
slogan> say_hello("hi"); 
-> Slogan says: hi
=> hi from C

The FFI can also deal with more complex C objects like pointers and structures. You can learn more by reading the Language Reference. Also see the FFI sample code that is in the repository.
