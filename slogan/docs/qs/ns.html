Modularity

A function can refer to variable bindings in the environment that created it. In the following program make_counter is a function that returns a function. The returned function maintains a reference to the argument of make_counter as its own "private state":

function make_counter(n) 
  ^(message) 
  | 'inc -> { n = n + 1; n }
  | 'dec -> { n = n - 1; n }
  | 'get -> n; 

let c1 = make_counter(10); 
let c2 = make_counter(100); 
c1('inc); 
//=> 11
c1('inc); 
//=> 12
c2('inc); 
//=> 101
c2('dec); 
//=> 100
c1('get); 
//=> 12

You can use the shorter dot-notation for "sending messages" to functions:

c2.inc; 
//=> 101
c2.inc; 
//=> 102
c2.get; 
//=> 102

Modules collect definitions into logical groups. The definitions that are visible outside the module are exported using a list of names similar to a function's parameter list.

module a(f,  x)
{ function f() "hello from module - a"; 
  let x = 100; };

Values exported from a module can be accessed using the `dot` notation:

a.f(); 
//=> hello from module - a
a.x; 
//=> 100
